## Relevant Excerpt

**METHONTOLOGY (Late 1990s)**  
Waterfall-style approach for single ontology development. Life cycle stages: Specification → Conceptualisation → Formalisation → Integration → Implementation → Maintenance. Management activities run in parallel across all stages: planning, control, quality assurance, knowledge acquisition, documentation, and evaluation. "METHONTOLOGY is one of the earliest and most widely used methodologies."

**NeOn Methodology (Post-2006)**  
Replaces linear waterfall with flexible scenario-based approach. Includes "55 activities" for collaborative, distributed development. Key innovation: emphasizes Competency Questions for specification, supports ontology networks and reuse/reengineering, addresses collaborative and distributed development. Nine scenarios cover everything from building from scratch to reusing, merging, and localizing existing ontologies.

**Ontology Development 101 (OD101)**  
Micro-level focus on formalization guidance; provides pragmatic authoring directives. Earlier methodology targeting individual practitioners. Reference: Noy & McGuinness, Stanford, 2001.

**Ontology Summit 2013 Lifecycle Model**  
Five phases: Requirements development → Ontological analysis → System design → Implementation → Deployment. Emphasizes answering specific questions at each stage; addresses evaluation criteria.
