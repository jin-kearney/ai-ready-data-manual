# 별첨 B — 코어 온톨로지 설계 기획서 (템플릿 + 완성 예시)

> 한 줄: **9단계 중 1~7단계(코어 설계)의 산출물 양식.** 빈 양식 채우기가 아니라 **구축 착수 가능한 실행안**이어야 한다. 코어는 유즈케이스와 무관하게 이 양식만 보고 설계한다.

> **쓰는 법:** 메인 가이드 [§6.2 코어 설계](../B-3%20온톨로지.md#sec62)를 읽으며 이 문서를 채운다. 설계 접근의 각론(노드를 어느 레벨로 등)은 [별첨 A 각론](B-3%20별첨%20A%20—%20유즈케이스%20온톨로지%20구축%20각론.md), 기술 검토(§5)는 메인 [§7 아키텍처](../B-3%20온톨로지.md#7-아키텍처기술-선택). 각 표는 `빈 템플릿` 행과 🏭 `전자BG CCL 완성 예시(예시 값)` 행을 함께 둔다 — 실제 PoC에서 실데이터로 대체.

**작성 완료 기준 (이게 다 채워져야 제출 완료):**

| 항목 | 완료 기준 |
|---|---|
| 도메인 목적 | 한 문장, 측정 지표 포함 |
| 노드 후보 | As-Is 분석서 근거 항목과 연결, 데이터 소스 확인 |
| 관계 목록 | 방향과 업무 의미 설명 포함 |
| 핵심 질문 | 노드·관계 경로로 답변 가능 여부 확인 |
| 기술 검토 | LPG/RDF 선택 근거·추론 시점 판단 포함 |
| 미결 사항 | 담당자·기한 포함 |

**As-Is 분석서 연동 (작성 전 확인):** As-Is 근거가 없는 노드·관계는 작성하지 않는다.

| As-Is 분석서 항목 | 이 기획서 반영 위치 |
|---|---|
| E2E 업무 흐름(VOC 접수~C/S 리포트) | §1 도메인 목적 적용 범위 |
| 현실에 존재하는 물리·조직·기준 객체 | §2 L1 마스터 객체 |
| 시간과 함께 일어나는 사건·이벤트 | §2 L2 사건 |
| 현업의 판단·귀책·대책 결과 | §2 L3 해석 |
| 데이터 소스 확인 결과(MES·ERP 등) | 각 노드 데이터 소스 컬럼 |
| 데이터 미확인·수기 항목 | "미확인" 표시 → §7 미결 |
| 팀 간 용어 불일치·동의어 | 노드 정의·alias 속성 |

## 4. 핵심 질문 및 경로 검증 (9단계 5단계)

코어가 답해야 할 질문 3~5개. 각 질문이 위 노드·관계로 경로 연결되는지 확인(미연결이면 설계 수정).

| # | 핵심 질문 | 노드·관계 경로(안) | 경로 연결 |
|---|---|---|---|
| 1 | 🏭 이 분기 들뜸의 상위 원인은? | `들뜸검출 ←EXPLAINS– 흡습판단/온도미달판단` 집계 | 연결 |
| 2 | 🏭 특정 프리프레그 Lot과 관련된 공정 파라미터는? | `프리프레그Lot –USED_IN→ 열압착집행{온도}` | 연결 |
| 3 | 🏭 지난 12개월 들뜸에 적용된 조치는? | `들뜸검출 ←REMEDIATES– 조치` | 연결 |
