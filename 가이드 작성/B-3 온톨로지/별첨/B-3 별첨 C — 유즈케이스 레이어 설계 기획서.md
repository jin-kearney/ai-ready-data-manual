# 별첨 C — 유즈케이스 레이어 설계 기획서 (템플릿 + 완성 예시)

> 한 줄: **9단계 중 8단계(유즈케이스 레이어)의 산출물 양식.** **코어를 재정의하지 않는다** — 코어([별첨 B](B-3%20별첨%20B%20—%20코어%20온톨로지%20설계%20기획서.md)) 위에서 특정 목적의 유즈케이스를 실행할 전용 구조만 설계하고, 코어에 이미 있는 것은 **참조**로만 연결한다.

> **쓰는 법:** 메인 가이드 [§6.3 유즈케이스 설계](../B-3%20온톨로지.md#63-유즈케이스-레이어-설계-8단계--산출물-유즈케이스-기획서)를 읽으며 채운다. 목적 유형별 설계 차이·암묵지 처리는 [별첨 A 각론](B-3%20별첨%20A%20—%20유즈케이스%20온톨로지%20구축%20각론.md). 🏭 완성 예시는 전자BG CCL "**C/S Report 초안 자동 생성**"(자동화형) — 모든 값은 예시.

**작성 완료 기준:** 유즈케이스 정의(목적·유형·사용자·수명) / 핵심 기능 질문(코어만으로 안 되는 이유 포함) / 입출력(형식·소스·대상) / 전용 노드·관계(코어 중복 없음, `layer` 태그) / 코어 참조 구조 / 코어 변경 요청(없으면 "없음") / 미결 사항.

## 7. 암묵지 구현 방식

| 단계 | 방법 | 🏭 CCL 산출물 |
|---|---|---|
| 수집 | 인터뷰·문서 | 클레임 묶기 기준(동일 Lot·기간) |
| 구조화 | 온톨로지 관계 vs 쿼리 로직 분류([별첨 A §2.2](B-3%20별첨%20A%20—%20유즈케이스%20온톨로지%20구축%20각론.md)) | 묶기=쿼리 로직, 위반판정=규칙 노드 |
| 검증 | SME 검수 | 클러스터 결과 표본 확인 |
| 적재 | 신규 클레임 시 클러스터 재계산 | — |

**7.2 상황별 구현 분기** (없으면 "해당 없음"): 🏭 클레임이 단건이면 클러스터 생략, 다건이면 묶음 — 그 외 해당 없음.

|---|---|---|

## 9. 설계 검증 (4원칙)

| 검증 축 | 검증 질문 | 🏭 CCL 결과 |
|---|---|---|
| 현실성 | 전용 노드가 실제 업무 흐름에서 확인되나 | 통과(클러스터=C/S 운영 실재) |
| 명시성 | 의미가 한 문장으로 설명되나 | 통과 |
| 재사용성 | 코어 중복 없이 다른 유즈케이스와 연결 여지 | 통과(코어 참조만) |
| 설명력 | 출력 이유를 관계 경로로 설명하나 | 통과(클러스터→공통Lot→위반→원인 경로) |
