## 정본 모델 (문서 전체 일관 사용)

- **구성 3요소:** ① 정답셋(Gold Set) ② 채점 기준(Rubric) ③ 전문가 코멘트(판단 근거·오류 유형). (§2.2·2.3·2.4)
- **만드는 절차 4단계:** ① 평가할 사례 모으기 → ② 정답셋 작성 → ③ 채점 기준 합의(노밍) → ④ 채점 연결. (§3.1)
- **채점 방법 3가지:** 코드 자동 매칭 / 사람 / AI 채점(LLM 평가). (§3.2) — §4.1 솔루션 유형 ①②③과 1:1 대응, ④ RAG·Agent 전용은 특화 묶음으로 명시(혼동 방지).
- **운영 3가지:** 회귀 테스트 / 실패 사례 추가 / 버전 관리. (§3.3) Evaluation Harness는 §3.3 콜아웃으로 "묶는 틀"로만 도입(별도 정본 아님).
- **러닝 예시:** 두산전자 CCL **Delamination** 품질 결함 원인 RAG (B-3·E-2와 동일 예시로 크로스가이드 일관). 분류 예시는 C/S Report. 설비 이상 원인 추천은 회귀 테스트 SVG. 수치는 모두 가상.
