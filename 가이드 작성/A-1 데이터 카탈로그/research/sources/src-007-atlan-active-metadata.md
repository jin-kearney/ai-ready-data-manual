## 능동 메타데이터(Active Metadata)란

정기적으로 수집되는 정적 문서가 아니라, 실시간 이벤트에 반응하여 지속 갱신되는 메타데이터.
2026년 시장 규모 $128.9억 달러로 예측.

## 수동(Passive) vs 능동(Active) 메타데이터 비교

| 차원 | 수동 방식 | 능동 방식 |
|------|----------|----------|
| 핵심 기능 | 정적 문서화 | 살아있는 인텔리전스 계층 |
| 수집 방식 | 수동 또는 배치 크롤링 | 자동·실시간 수집 |
| 갱신 주기 | 주기적/반응적 | 지속적/선제적 |
| 시스템 상호작용 | 데이터 기술만 | 관찰·학습·조치 |
| 변경 대응 | 장애 후 감지 | 변경 즉시 감지 |

## 지속 추적하는 메타데이터 유형

1. **기술 메타데이터**: 스키마 변경, 컬럼 정의, 데이터 타입
2. **운영 메타데이터**: 쿼리 로그, 사용 패턴, 리니지 이벤트
3. **비즈니스 메타데이터**: 오너십 기록, 분류, 글로서리 항목
4. **협업 신호**: 팀 대화, 문서화 업데이트
5. **품질 지표**: 완전성, 정확성, 최신성, 이상 징후
6. **접근 패턴**: 인기도 추이, 사용자 상호작용, 의존성

## AI-Ready 데이터 발견 지원

능동 메타데이터는 AI 에이전트가 검증된 컨텍스트 기반으로 현재의 거버넌스 정보를 조회하도록 보장.

3가지 핵심 속성:
1. **실시간 전파**: 스키마 변경·정책 갱신이 수분 내 연결 시스템에 반영
2. **양방향 API 접근**: 에이전트가 메타데이터를 읽고 관찰 내용을 기록
3. **기본 거버넌스 컨텍스트**: 분류 레이블·접근 권한이 자동 삽입

## 기업 맥락 계층(Context Layer)에서의 역할

능동 메타데이터는 AI 에이전트에게 신뢰할 수 있고 거버넌스된 기업 지식에 실시간 접근을 제공하는
인프라 스택의 핵심 구성 요소.
