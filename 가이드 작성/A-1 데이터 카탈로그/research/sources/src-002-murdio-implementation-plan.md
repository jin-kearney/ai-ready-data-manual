## 4단계 구현 프레임워크

### Phase 1: 범위 설정·비즈니스 케이스 (1~6주)

**목표:** 기술 도구 도입 전, 프로그램 성공 조건 수립.

주요 활동:
- 단일 비즈니스 Pain 기반 사용 사례 정의 (규정 준수 리니지, KPI 문서화, 도메인 오너십 모델링)
- 경영진이 책임지는(단순 지원이 아닌) 스폰서 확보
- "Phase 1 완료" 기준을 구체적 성과 지표로 문서화
- 세 가지 합의 문서화: ① 무엇을 해결하는가, ② 누가 책임지는가, ③ 성공이란 무엇인가

### Phase 2: 플랫폼 선정·팀 구성 (4~10주)

**플랫폼 선정 기준:**
- 사용 사례 적합성, 메타데이터 모델 유연성, 통합 생태계 호환성
- 거버넌스 워크플로우 지원, 확장성, TCO, 벤더 지원

**핵심 팀 구성:**

| 역할 | 책임 | 시기 |
|---|---|---|
| 경영진 스폰서 | 비즈니스 책임, 예산, 에스컬레이션 | Phase 1 |
| 데이터 거버넌스 리드 | 전략, 정책, 스튜어드십 모델 | Phase 1 |
| 프로젝트 매니저 | 일정, 조율, 이해관계자 소통 | Phase 1 |
| 데이터 카탈로그 관리자(IT) | 플랫폼 설정, 구성, 통합 | Phase 2 |
| 데이터 스튜어드(비즈니스) | 자산 오너십, 메타데이터 큐레이션, 워크플로우 | Phase 2~3 |
| 도메인 데이터 오너 | 도메인 내 데이터 품질 책임 | Phase 3 |

### Phase 3: 파일럿 배포 (8~20주)

**핵심 원칙:** "하나의 도메인, 하나의 사용 사례, 높은 가시성. 범위 확장 압박은 모두 거부하라."

구성 활동:
- 파일럿 사용 사례에만 카탈로그 구조 설정
- 고가치 데이터셋만 수집 (전체 수집 X)
- 범위 내 자산에 오너십 배정
- 기본 워크플로우 수립: 오너십 배정, 인증, 용어집 라이프사이클
- 액세스 통제 및 사용자 역할 구성

**스튜어드 참여:**
- 발표가 아닌 실습 세션
- 스튜어드가 실제 거버넌스 작업을 카탈로그에서 직접 수행
- 전 과정 적극 피드백 수집

**성공 검증 지표:**
- 대상 도메인 자산 커버리지 %
- 오너십 커버리지 %
- 스튜어드 참여 완료율
- 데이터 발견 시간 개선

### Phase 4: 전사 롤아웃·채택 (6~18개월+)

**도메인 확장 전략:**
- 비즈니스 임팩트·이해관계자 준비도 기준 우선순위
- 각 신규 도메인마다 파일럿 패턴 반복
- 여러 도메인 동시 온보딩 금지

**자동화 구현 (거버넌스 모델 안정화 후):**
- 원천 시스템의 메타데이터 자동 수집
- 예약된 신선도(freshness) 점검 및 리니지 업데이트
- PII·민감 데이터 분류 자동 태깅
- API를 통한 대량 업데이트
- 데이터 파이프라인 이벤트 연동 워크플로우 트리거

## 구현 전 전제조건
1. 명확히 정의된 사용 사례 (구체적 비즈니스 Pain)
2. 결과에 책임지는 경영진 스폰서
3. 데이터 도메인 매핑 완료 (개략적이라도)
4. 초기 배정 포함 데이터 오너십 모델 시작
5. 경량 거버넌스 정책·프레임워크
6. IT 인프라 준비 상태 평가 (클라우드 vs 온프레미스, SSO, 통합 환경)
7. 비즈니스·IT 연결하는 내부 프로젝트 리드

## 주요 성과 지표 목표치

| 지표 | 목표 기준 |
|---|---|
| 카탈로그 채택률 | 6개월 시점 월간 활성 사용자 >60% |
| 자산 커버리지 | 파일럿 완료 시 우선 도메인 >70% |
| 데이터 오너십 커버리지 | 활성 도메인 >80% |
| 스튜어드 작업 완료율 | SLA 내 >75% |
| 용어집 채택률 | 핵심 비즈니스 용어 >50% 연결 |

## 흔한 실패 패턴
1. **문제 정의 전 도구부터 도입** → 아무도 안 쓰는 고비용 메타데이터 저장소
2. **출시 전 데이터 오너십 미확보** → 수개월 내 카탈로그 노후화
3. **전체 동시 카탈로깅 시도** → 범위 비대로 운영 단계 진입 실패
4. **변경 관리 과소평가** → 조직 워크플로우 변화 없는 기술 배포
5. **실질적 책임 없는 경영진 스폰서** → 경쟁 우선순위 발생 시 표류

## 타임라인 요약
- 범위 설정·비즈니스 케이스: 1~6주
- 플랫폼 선정·팀 구성: 4~10주 (Phase 1과 중첩)
- 파일럿 배포: 8~20주
- 전사 롤아웃·채택: 6~18개월+
- 대기업 전체 구현: 9~18개월
