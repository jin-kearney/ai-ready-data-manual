# F-2 데이터 생애주기 관리 — 작업 메모 (주제 로컬 규칙·결정 이력)

## 주제 기본
- 코드/명: F-2 데이터 생애주기 관리(Data Lifecycle Management)
- 정의: 데이터의 생성·사용·보관·아카이빙·폐기까지의 시간축을 정책 기반으로 통제하는 체계.
- 유형: **기본형**(다 필요) → Why·When 분리. 거버넌스(F) 그룹, "시간축(수명)" 담당.
- 계열사 예시: 두산퓨얼셀(발전용·건물용 수소연료전지) — 스택 내구 시험·제조 이력·품질 성적서·C/S 리포트(전부 가상 예시).
- KPI·로드맵 섹션 없음(KPI는 C군만, 로드맵은 전 주제 제거). Where가 마지막 섹션.

## 정본 모델·결정 (2026-06-26 초안)
- **생애주기 5단계 + 재사용 루프**: 생성·수집 → 사용(Active) → 보관(Inactive) → 아카이빙 → 폐기, 아카이빙→사용 복원 루프. DAMA-DMBOK 6단계를 제조 현장용으로 단순화. 이 다섯 단계 명칭을 문서 끝까지 일관 사용.
- **보존 기간 결정 5기준**: ① 법적·규제 의무 ② 업무 재사용 가치 ③ AI 학습 가치 ④ 감사 필요성 ⑤ 저장 비용·위험. "가장 긴 요건에 맞춤". 결정 흐름은 mermaid 의사결정 트리.
- **저장소 계층 4단**: Hot(운영) / Warm(보관) / Cold(아카이브) / 폐기·만료 대기. 자동 티어링이 KQ5(비용·위험)의 핵심. → 커밋 SVG `이미지/저장소-계층-사다리.svg`.
- **과정 데이터(KQ3)**: 불량·실패·재작업은 희소 학습 자산 → "보존". 제조업 특유 판단으로 §2.4 독립.
- **역할 4갈래**: 데이터 오너 / 스튜어드 / 커스터디언 / 법무·컴플라이언스. (RBAC 등 약어 안 씀, 평어로.)
