## 정본 모델 (끝까지 일관 유지)

- **데이터 계통이란**: 데이터가 어디서 와서(출처) 어떤 가공을 거쳐(변환) 어디 쓰였는지(활용)를 잇는 경로 기록. 구조는 방향 그래프(DAG) — 노드=데이터 자산, 엣지=변환·이동.
- **계통이 기록하는 세 가지**(§2.3, 정본): ① 연결(경로) ② 변환 이력 ③ AI 답변 근거 연결(Citation). 가이드 전체가 이 셋을 일관 재사용.
- **입도 3수준**: 테이블 / 컬럼(Column-level) / 청크·근거. 영향도·감사엔 컬럼 단위 필요.
- **계통으로 답하는 두 질문**(§2.4): 하류=영향도 분석("바뀌면 뭐가 깨지나"), 상류=근본 원인·감사("왜·어디서 왔나").
- **수집 방식 3가지**(§6.2): 정적 파싱 / 런타임 이벤트(OpenLineage) / 수기 선언. 섞어 쓰되 자동 수집 기본, 수기는 공백만.
- **경계**: 실시간 이상=C-1 / 사용 판정=C-2 / 보존·폐기=F-2 / 위치정보 부착=B-1 / Tool 호출 로그=D-2.

## 결정사항·주의점

- **KQ 박스 미삽입**: 최신 형제 가이드(B-1 v0.21·B-3)가 핵심질문 박스를 제거한 깨끗한 흐름이라 동일하게 맞춤. KQ는 위 표로 자가 점검만(독자용 박스 X). 마스터 프롬프트 0623 "요약박스 금지"와도 정합.
- **SVG 4종 신규 작성**(`이미지/`): 계통-추적-그래프(hero, §4)·테이블-컬럼-계통(§2.2)·AI답변-근거카드(§2.3)·영향도분석-상하류(§2.4). B-1의 "모형 SVG" 방식 그대로(흰 배경·파랑6/초록3·주석 리더선). 사용자 요청("svg 그림들로 표현").
- **Mermaid 6종**: 체계 내 위치(2.1)·정본 흐름 chain(2.2)·When quadrant(3)·구축 절차(6.1)·운영 루프(6.4)·조감도(7). 02 표준 컬러·테마 헤더 준수.
- **Tech Stack 1층/2층**: 5장은 계통 기능만 비교(1층). 2층 정본(`01 Tech Stack 비교`)에 C-3 열·OpenLineage+Marquez 행 이미 존재 → 링크만, 중복 안 함. **정본 반영은 사용자 명시 승인 시에만**(현재 보류).
- **데이터 준비 관점 유지**: LLM 관측 도구(LangSmith·Langfuse)는 "모델 만들기"가 아니라 "답변 근거 데이터를 기록하는 장치"로만 다룸. RAG/Agent 구현법 아님.
- **레퍼런스 번호**: Spline은 [7s](2.4의 [7] Atlan 영향도와 충돌 회피). 전 마커-앵커 1:1 정합 확인.
