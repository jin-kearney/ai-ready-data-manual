# E-2 합성데이터 (Synthetic Data)

## 1. 개요

합성데이터는 실데이터가 부족하거나 사용이 제한되는 상황에서 AI 학습·검증을 위해 인공적으로 생성한 데이터이다.

합성데이터의 목적은 실데이터를 완전히 대체하는 것이 아니라, 데이터 부족 문제를 보완하고 AI 활용 범위를 확대하는 것이다.

본 가이드는 제조업 환경에서 어떤 경우에 합성데이터를 활용해야 하며, 어떤 방식으로 생성·검증·운영해야 하는지를 설명한다.

---

## 2. 왜 필요한가 (Why)

### 실데이터 부족

- 불량 사례가 매우 적음
- 희귀 장애가 거의 발생하지 않음
- 신규 설비는 학습 데이터가 없음

### 보안·규제 문제

- 개인정보 포함
- 고객 데이터 활용 제한
- 외부 협력사 제공 불가

### 테스트 데이터 부족

- 운영 환경 재현 어려움
- 예외 상황 검증 어려움
- Agent 시뮬레이션 데이터 부족

### AI-ready Data 관점

합성데이터는 부족한 데이터를 보완하여 AI 학습·검증이 가능한 상태를 만드는 데이터 자산화 활동이다.

---

## 3. 무엇을 갖추나 (What)

### 합성 대상 데이터

- 품질 불량 데이터
- 설비 이상 데이터
- 공정 센서 데이터
- 고객 클레임 데이터
- 테스트 데이터

### 생성 방식

- 규칙 기반
- 통계 기반
- 시뮬레이션 기반
- 생성형 AI 기반

### 검증 체계

- 분포 검증
- 품질 검증
- 활용성 검증
- 재식별 위험 검증

### 운영 체계

- 생성 이력
- 버전 관리
- Synthetic Tag
- 사용 정책

---

## 4. 어디부터 적용하나 (Where)

### 우선 적용 대상

- 희귀 불량
- 설비 고장
- 장애 이벤트
- 신규 제품
- 신규 공정

### 적용하지 않아도 되는 경우

- 충분한 실데이터 존재
- 데이터 수집 비용 낮음
- 실제 데이터 사용 제약 없음

---

## 5. 어떻게 구축·운영하나 (How)

### 정본 모델

```text
목적 정의
→ 보존 특성 정의
→ 생성 방식 선택
→ 합성데이터 생성
→ 검증 및 위험 관리
→ 운영 및 자산화
```

### 단계 1. 목적 정의

품질 예측 모델 학습인지, 설비 이상 탐지인지, 테스트 데이터 생성인지 목적을 정의한다.

### 단계 2. 보존 특성 정의

합성 과정에서 반드시 유지해야 할 특성을 정의한다.

예시

- 평균
- 분산
- 상관관계
- 시계열 패턴
- 결함 비율

### 단계 3. 생성 방식 선택

#### 통계 기반

적용

- 정형 데이터
- 단순 분포 재현

예시

- Copula
- Bayesian Network

#### 생성모델 기반

적용

- 복잡한 다변량 데이터
- 비정형 데이터

예시

- CTGAN
- TVAE

#### 시뮬레이션 기반

적용

- 물리 현상
- 설비 동작

### 단계 4. 합성데이터 생성

품질 데이터셋을 기반으로 합성데이터 생성

예시

- Delamination
- Wicking
- Void

불량 데이터 부족 보완

### 단계 5. 검증 및 위험 관리

검증 항목

- 분포 유사성
- 모델 성능 유지
- 이상치 검증
- 개인정보 재식별 위험

Synthetic Tag 부여

### 단계 6. 운영 및 자산화

생성 결과를 재사용 가능한 데이터 자산으로 등록한다.

- 데이터 카탈로그 등록
- 메타데이터 등록
- 버전 관리
- 재사용 이력 관리

#### Phase 4. Assetization

- 전사 합성데이터 자산화
- 계열사 확산
