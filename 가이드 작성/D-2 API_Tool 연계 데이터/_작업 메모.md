# D-2 API/Tool 연계 데이터 — 작업 메모

이 주제의 결정·정본 모델·주의점·피드백을 누적 기록한다. (전역 규칙은 `CLAUDE.md`·`공통 규칙/`)

## 정본 모델 (끝까지 일관 유지)

- **한 줄 정의:** AI 에이전트가 외부 시스템·Tool을 안전하게 호출하도록 기능·입출력·제약을 정의한 **명세 체계**.
- **정본 프레이밍:** D-2는 *에이전트를 만드는 법*이 아니라 *에이전트가 읽을 **Tool 명세를 데이터로 준비하는 법***. (CLAUDE.md·메모리 `data-prep-not-build` — D-2 특별 주의 항목)
- **★ 스토리라인 축(2026-07-02 고객 확정):** "회사에 이미 있는 API를 에이전트가 읽는 MCP Tool 명세 데이터로 바꾸는 여정". 명세 구성·표준·MCP화 방법론이 본론.
- **Tool 명세 한 건 = 2구성:** ① 설명서(AI가 읽고 고르는 설명 — 기능·목적·입력·출력·제약·호출예시, §2.2) ② 입출력 규격(기계가 검증하는 약속 — 필수·타입·허용값·응답·오류, §2.3). **"Tool 후보 목록"은 산출물이 아니라 선정 작업 → How ①(§3.2)** (B-2가 "① 대상 선정"을 How에 두는 것과 같은 이치 — 2026-07-02 결정, What에 두면 레벨이 안 맞음).
- **위험도 4등급(표준값):** 조회형 → 추천형 → 실행형 → 외부 전송형. 게이트 아래(조회·추천)=자동, 위(실행·외부전송)=사람 승인. (§1.4)
- **구축 5단계(개정):** ① Tool 후보 선정 → ② 명세 작성(사람용 문서→AI용) → ③ MCP화(변환·검증·등록) → ④ 운영—버전·오류 → ⑤ 운영—호출 기록. (§3)
- **명세 표준 3종 = What(§2.4):** JSON Schema(약속 언어) ⊂ OpenAPI(REST API 기술) · MCP(에이전트↔Tool 프로토콜). MCP tool 정의 = 이름+설명(§2.2)+입력 스키마(§2.3) — "2장 내용물을 담는 표준 그릇" 프레임.
- **용어 통일:** "Tool"(도구는 1회 병기) · "명세" · "에이전트(AI Agent)" · "등록소(Registry)". 약어는 최초 1회 병기.

## 주의점 / 결정

- **상단 KQ 박스는 두지 않음** — B-3(v0.30 제거)·B-1 현행 컨벤션과 00 목차 "B-3 기준" 지시를 따름. KQ는 위 매핑으로 자가 점검(합격 기준은 커버리지지 박스 유무가 아님).
- **(2026-07-02 고객) 독립 "Standard" H2 금지** — 표준 지식(뭐가 있나)은 What, 표준 형식으로 만들고 취합하는 법은 How. 예시 시나리오 독립 섹션도 금지 — 전역 규칙대로 전/후→Why 끝, 흐름→How 콜아웃으로 흡수.
- **§3.4 MCP화 이탈 경계:** MCP 서버 구축·배포 코드는 다루지 않는다 — "기존 API 문서에서 무엇을 빼고·다시 쓰고·정리해야 에이전트가 읽는 데이터가 되나"(OpenAPI→MCP 매핑 표)가 본체. 자동 변환 도구를 써도 description 재작성 필수·①에서 고른 것만 노출.
- **데이터 준비 관점 이탈 경계:** §3.1 흐름 sequence는 "명세를 갖추면 이렇게 돈다"는 *효과 미리보기*로만(에이전트 오케스트레이션 설명 금지). 호출 순서·판단=D-3, 로그 활용 개선=E-4, 권한·승인 거버넌스=F-4로 포인터.
- **계열사:** 현장 지원 에이전트 E2E 1개 thread로 §1.2·§3 일관(getRobotAlarms 조회형 + createWorkOrder 실행형). **본문 표기는 계열사명 없이 "협동로봇 A/S" 등 제품·공정 표현만**(전역 규칙 `guide-examples-no-affiliate-name`, v0.2에서 두산로보틱스 표기 제거).
- **출처:** MCP는 빠르게 변함 → 버전·일자 "PoC 확인" 단정 회피. OWASP는 공식 번호 LLM06(=Excessive Agency, LLM08 아님). OpenTelemetry 정본은 github semantic-conventions-genai(옛 opentelemetry.io 경로 이동됨).
- **Tech Stack 정본 reflect-back:** 미실시(사용자 명시 요청 시에만). 새 솔루션(MCP Registry·Kong·Apigee·AWS API GW)은 승인 후 정본 Part A에 반영.
