# C-2 데이터 품질 관리 — 작업 메모

이 주제 전용 규칙·결정·주의점. 가이드 작성/수정 전 먼저 읽는다.

## 정본 모델 (끝까지 일관 유지)

- **품질 6차원**: 완전성·정확성·일관성·최신성·유효성·유일성 (DAMA DDQ 근거 [\[2\]]). 표준별로 더 잘게 나누지만(DMBOK 8개·ISO/IEC 25012 15개) 현업 눈높이로 6개 고정. 본문·SVG·항목 사전·KPI 전부 이 6개로 통일.
- **품질 = 용도 적합성(Fitness for Use)**: 절대 기준이 아니라 "이 AI 용도에 쓸 수 있는가" 상대 판정 → 차원별 합격선은 업무·용도별로 다르게.
- **Quality Gate 한 모델**: "통과해야만 들어간다." 구현 방식(적재-점검-공개 WAP·회로 차단·데이터 계약)은 한 줄씩만 풀이하고 약어·구현 디테일은 줄임.
- **예외 승인 4기록**: 승인자·사유·기간·범위. 영구 허가 아닌 한시 통과.
- **KPI 정본 3개**: 품질 기준 통과율(↑)·품질 미달 차단 건수(모니터)·품질 개선 리드타임(↓). 탐지 시간(MTTD)은 C-1으로 넘김.

## 구조 (00 전체 목차 C-2 = 9섹션, C군이라 KPI 유지·로드맵 제외)

- 순서 주의: **Quality Gate(§3)가 When 앞**, **Tech Stack(§6)이 How 앞** — 00 목차 그대로.
- KQ→섹션: KQ1 품질기준→§2.2(kq1) / KQ2 사용금지→§2.3(kq2) / KQ3 게이트→§3.1(kq3) / KQ4 예외승인→§3.2(kq4) / KQ5 측정→§9(kq5). 5개 전부 커버.
- 로드맵·미래 전망은 두지 않음(00 목차 2026-06 결정: 20개 주제 전부 제거). C군이라 KPI만 마지막에.

## 경계 (MECE)

- C-1 Observability = 운영 중 실시간 이상 감지 / **C-2 = 투입 전 합격 판정(게이트)**. 분포·이상치가 접점.
- C-3 Lineage = 사후 이력 추적. 출처 불명 판정 때 C-3 참조.
- F-4 = 접근권한·동의·민감정보. 같은 관문에서 함께 걸리되 **품질(C-2)과 권한(F-4) 책임 분리**. EU AI Act는 양쪽 걸침 → "거버넌스 요건"으로 한 번만 언급.
- **관측 솔루션(Monte Carlo·Anomalo·Bigeye 등)은 C-1과 같은 제품군** — C-2는 게이트로, C-1은 모니터링으로 쓴다고 §6에 경계 명시. 묶음 비교는 2층 정본으로 링크.
