# 별첨 C. 운영 기준서 — 신뢰성(C-1·C-2·C-3) 공통

신뢰성 세 주제 — [C-1 Observability](../C-1%20Observability/C-1%20Observability.md)(관측), [C-2 데이터 품질 관리](../C-2%20데이터%20품질%20관리/C-2%20데이터%20품질%20관리.md)(품질), [C-3 데이터 계통 Lineage](../C-3%20데이터%20계통%20Lineage/C-3%20데이터%20계통%20Lineage.md)(계통) — 은 같은 운영 기준 위에서 돌아간다. 누가 책임지고(오너십), 누가 무엇을 하고(역할·책임), 데이터 생애 단계별로 무엇을 지키는지(생애주기)를 세 주제가 공유하므로, 여기서 한 번만 정의한다. 각 가이드는 자기 KPI만 다루고 운영 기준은 이 문서를 참조한다.

운영 기준은 "이렇게 운영하겠다"는 약속이다. 아래 세 가지 — 데이터 오너십·역할과 책임(R&R)·생애주기(Lifecycle) — 를 관리 대상마다 정의하고, 정의 여부는 [4장](#s4)의 점검 지표로 확인한다.

<a id="s1"></a>

## 1. 데이터 오너십 (Data Ownership)

관리 대상 데이터마다 **데이터 오너 1명**을 지정한다. 데이터 오너는 그 데이터를 실제로 생성·사용하는 현업 부서에서 도메인(제품군·공정·업무) 단위로 맡는다. 도구·플랫폼을 다루는 데이터팀이 아니라, 데이터의 내용과 기준을 아는 현업이 오너가 된다.

데이터 오너의 공통 책임은 다음과 같다.

- 그 데이터의 **정상/합격 기준을 승인**한다(무엇이 정상 범위이고, 어느 수준이면 AI에 써도 되는가).
- 이상·미달이 확인되면 **최종 판단**을 내린다(조치·차단·예외 활용 여부).
- 관리 대상 **등록·해제를 승인**한다.

주제별로 오너가 승인·판단하는 대상은 다르다.

| 주제 | 데이터 오너가 승인·판단하는 것 |
|------|------|
| C-1 관측 | 정상 범위(한계값·추세) 승인, 이상 발생 시 업무 조치 여부 |
| C-2 품질 | 품질 합격선 승인, 합격선 미달 데이터의 활용 여부 |
| C-3 계통 | 계통 정보(출처·변환·활용 경로)의 정확성, 계통 단절·오류의 수정 우선순위 |

<a id="s2"></a>

## 2. 역할·책임 (R&R)

신뢰성 관리의 책임은 세 계층으로 나눈다. 계층은 세 주제가 공통이며, 계층별로 하는 일이 주제마다 구체화된다.

| 역할 계층 | 공통 책임 | C-1 관측 | C-2 품질 | C-3 계통 |
|------|------|------|------|------|
| **데이터 오너** (현업 부서) | 기준 승인·최종 판단 | 정상 기준 승인, 이상 시 조치 판단 | 합격선 승인, 미달 활용 판단 | 계통 정확성 책임, 수정 우선순위 판단 |
| **운영 담당** (데이터·플랫폼팀) | 규칙·도구 설정과 일상 운영 | 관측 규칙·임계값 설정, 알림 운영 | 품질 규칙 정의, Quality Gate 운영 | 계통 수집·갱신, 근거(Citation) 기록 운영 |
| **검증·처리 담당** | 정합성 점검과 예외·복구 처리 | 멈춘 파이프라인 복구·재처리(IT 운영) | 규칙 검증, 예외 활용 승인(품질 책임자) | 계통 정합성·근거 추적 가능 여부 점검 |

한 사람이 여러 계층을 겸할 수 있으나, **기준을 승인하는 사람(오너)과 규칙을 운영하는 사람(운영 담당)은 구분**한다 — 만든 사람이 스스로 합격을 주지 않도록 한다.

<a id="s3"></a>

## 3. 생애주기 (Lifecycle)

데이터가 **수집→적재→활용→보관/폐기** 단계를 거치는 동안 신뢰성 관리를 유지한다. 단계별로 무엇을 지키는지는 주제마다 다르다.

| 생애 단계 | C-1 관측 | C-2 품질 | C-3 계통 |
|------|------|------|------|
| 수집·적재 | 신규·변경 데이터는 평소 추세가 쌓일 때까지 기준선 학습 기간을 따로 둔다 | 신규 제품·공정이 생기면 품질 합격선을 새로 정한다 | 원천부터 적재까지의 경로를 계통으로 등록한다 |
| 활용 | 운영 중 상시 관측하고 이상 시 알린다 | AI 투입 전 Quality Gate로 판정한다 | 활용·AI 답변의 근거(Citation)를 기록한다 |
| 보관·폐기 | 폐기 예정 데이터는 관측 대상에서 정리한다 | 기준이 현실과 어긋나면 오너 합의로 갱신한다 | 근거·이력을 보존 기간까지 유지한 뒤 폐기한다 |

보관·폐기의 기간과 정책 자체는 [F-2 데이터 생애주기 관리](../F-2%20데이터%20생애주기%20관리/F-2%20데이터%20생애주기%20관리.md)를 따르고, 멈춘 파이프라인의 복구·재처리 운영은 [F-1 데이터 운영관리](../F-1%20데이터%20운영관리/F-1%20데이터%20운영관리.md)가 맡는다. 이 운영 기준서는 신뢰성 관점의 유지 기준만 정한다.

<a id="s4"></a>

## 4. 운영 기준이 자리 잡았는지 점검

운영 기준이 실제로 적용됐는지는 **운영 기준 정의율** 하나로 세 주제 공통으로 점검한다.

- 정의: 관리 대상 중 데이터 오너십·R&R·생애주기 기준이 정의된 비율
- 산식: 운영 기준 정의 대상 / 전체 관리 대상
- 목표: 핵심 관리 대상부터 100%를 목표로, 도메인별로 단계 확대

각 주제의 성과 지표(KPI)는 해당 가이드의 KPI 절에 있다 — [C-1](../C-1%20Observability/C-1%20Observability.md#kpi) · [C-2](../C-2%20데이터%20품질%20관리/C-2%20데이터%20품질%20관리.md#kpi) · [C-3](../C-3%20데이터%20계통%20Lineage/C-3%20데이터%20계통%20Lineage.md#kpi).
