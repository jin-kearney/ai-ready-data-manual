# 별첨 C. 운영 기준서 — 신뢰성(C-1·C-2·C-3) 공통

신뢰성 세 주제 — [C-1 Observability](../C-1%20Observability/C-1%20Observability.md)(관측), [C-2 데이터 품질 관리](../C-2%20데이터%20품질%20관리/C-2%20데이터%20품질%20관리.md)(품질), [C-3 데이터 계통 Lineage](../C-3%20데이터%20계통%20Lineage/C-3%20데이터%20계통%20Lineage.md)(계통) — 은 같은 운영 기준 위에서 돌아간다. 누가 책임지고(오너십), 누가 무엇을 하고(역할·책임), 데이터 생애 단계별로 무엇을 지키는지(생애주기)를 세 주제가 공유하므로, 여기서 한 번만 정의한다. 각 가이드는 자기 KPI만 다루고 운영 기준은 이 문서를 참조한다.

운영 기준은 "이렇게 운영하겠다"는 약속이다. 아래 세 가지 — 데이터 오너십·역할과 책임(R&R)·생애주기(Lifecycle) — 를 관리 대상마다 정의하고, 정의 여부는 [4장](#s4)의 점검 지표로 확인한다.

<a id="s1"></a>

## 1. 데이터 오너십 (Data Ownership)

관리 대상 데이터마다 **데이터 오너 1명**을 지정한다. 데이터 오너는 그 데이터를 실제로 생성·사용하는 현업 부서에서 도메인(제품군·공정·업무) 단위로 맡는다. 도구·플랫폼을 다루는 데이터팀이 아니라, 데이터의 내용과 기준을 아는 현업이 오너가 된다.

데이터 오너의 공통 책임은 다음과 같다.

- 그 데이터의 **정상/합격 기준을 승인**한다(무엇이 정상 범위이고, 어느 수준이면 AI에 써도 되는가).
- 이상·미달이 확인되면 **최종 판단**을 내린다(조치·차단·예외 활용 여부).
- 관리 대상 **등록·해제를 승인**한다.
- 누가 그 데이터를 볼 수 있는지(**접근 권한**)를 승인한다 — 접근 권한·민감정보 통제의 실제 규칙은 [F-4 AI 데이터 권한 보안](../F-4%20AI%20데이터%20권한%20보안/F-4%20AI%20데이터%20권한%20보안.md)을 따른다.

주제별로 오너가 승인·판단하는 대상은 다르다.

| 주제 | 데이터 오너가 승인·판단하는 것 |
|------|------|
| C-1 관측 | 정상 범위(한계값·추세) 승인, 이상 발생 시 업무 조치 여부 |
| C-2 품질 | 품질 합격선 승인, 합격선 미달 데이터의 활용 여부 |
| C-3 계통 | 계통 정보(출처·변환·활용 경로)의 정확성, 계통 단절·오류의 수정 우선순위 |

<a id="s2"></a>

## 2. 역할·책임 (R&R)

신뢰성 관리의 책임은 세 계층으로 나눈다. 계층은 세 주제가 공통이며, 계층별로 하는 일이 주제마다 구체화된다.

| 역할 계층 | 공통 책임 | C-1 관측 | C-2 품질 | C-3 계통 |
|------|------|------|------|------|
| **데이터 오너** (현업 부서) | 기준 승인·최종 판단 | 정상 기준 승인, 이상 시 조치 판단 | 합격선 승인, 미달 활용 판단 | 계통 정확성 책임, 수정 우선순위 판단 |
| **운영 담당** (데이터·플랫폼팀) | 규칙·도구 설정과 일상 운영 | 관측 규칙·임계값 설정, 알림 운영 | 품질 규칙 정의, Quality Gate 운영 | 계통 수집·갱신, 근거(Citation) 기록 운영 |
| **검증·처리 담당** | 정합성 점검과 예외·복구 처리 | 멈춘 파이프라인 복구·재처리(IT 운영) | 규칙 검증, 예외 활용 승인(품질 책임자) | 계통 정합성·근거 추적 가능 여부 점검 |

한 사람이 여러 계층을 겸할 수 있으나, **기준을 승인하는 사람(오너)과 규칙을 운영하는 사람(운영 담당)은 구분**한다 — 만든 사람이 스스로 합격을 주지 않도록 한다.

복구·재처리와 변경 관리의 **실제 실행 절차**(Runbook·롤백·변경 승인)는 [F-1 데이터 운영관리](../F-1%20데이터%20운영관리/F-1%20데이터%20운영관리.md)가 정한다. 이 운영 기준서는 누가 책임지는지(R&R)까지만 정한다.

<a id="s3"></a>

## 3. 생애주기 (Lifecycle)

데이터가 **수집→적재→활용→보관/폐기** 단계를 거치는 동안 신뢰성 관리를 유지한다. 단계별로 무엇을 지키는지는 주제마다 다르다.

| 생애 단계 | C-1 관측 | C-2 품질 | C-3 계통 |
|------|------|------|------|
| 수집·적재 | 신규·변경 데이터는 평소 추세가 쌓일 때까지 기준선 학습 기간을 따로 둔다 | 신규 제품·공정이 생기면 품질 합격선을 새로 정한다 | 원천부터 적재까지의 경로를 계통으로 등록한다 |
| 활용 | 운영 중 상시 관측하고 이상 시 알린다 | AI 투입 전 Quality Gate로 판정한다 | 활용·AI 답변의 근거(Citation)를 기록한다 |
| 보관·폐기 | 폐기 예정 데이터는 관측 대상에서 정리한다 | 기준이 현실과 어긋나면 오너 합의로 갱신한다 | 근거·이력을 보존 기간까지 유지한 뒤 폐기한다 |

보존 기간을 정하는 기준(법적 의무·업무 가치·AI 재사용 가치·비용)과 폐기·아카이빙 **정책 자체**는 [F-2 데이터 생애주기 관리](../F-2%20데이터%20생애주기%20관리/F-2%20데이터%20생애주기%20관리.md)가 정본이다. 이 운영 기준서는 그 정책 위에서 신뢰성 관점의 유지 기준(무엇을 관측·판정·기록 상태로 둘지)만 정한다.

<a id="s4"></a>

## 4. 운영 기준이 자리 잡았는지 점검

운영 기준이 실제로 적용됐는지는 **운영 기준 정의율** 하나로 세 주제 공통으로 점검한다.

- 정의: 관리 대상 중 데이터 오너십·R&R·생애주기 기준이 정의된 비율
- 산식: 운영 기준 정의 대상 / 전체 관리 대상
- 목표: 핵심 관리 대상부터 100%를 목표로, 도메인별로 단계 확대

<a id="s5"></a>

## 5. 성과 지표 (KPI) — 세 주제 통합

각 주제가 상시로 보는 핵심 KPI를 한자리에 모았다. 목표치는 절대값을 미리 못 박지 않고, 현재 수준(기준선)을 측정한 뒤 데이터 오너와 목표치·기한을 SLA로 합의해 목표 대비로 관리한다.

| 주제 | KPI | 무엇을 재나 | 측정 (산식) |
|------|------|-------------|-------------|
| **C-1 관측** | 관측 커버리지 | 핵심 데이터 흐름 중 관측이 적용된 비율 | 관측 등록 파이프라인 / 전체 핵심 파이프라인 |
| | 이상 감지 시간(MTTD) | 이상 발생부터 알림까지 걸린 평균 시간 | 평균(알림 시각 − 이상 발생 시각) |
| | 오탐률 | 전체 알림 중 실제 이상이 아닌 비율 | (전체 알림 − 실제 이상) / 전체 알림 |
| **C-2 품질** | Quality Gate 적용률 | AI 활용 데이터 중 게이트가 적용된 비율 | 게이트 적용 데이터 / AI 활용 데이터 |
| | Quality Gate 통과율 | 게이트를 통과한 비율(적정 범위로 관리) | 통과 데이터 / 검증 데이터 |
| | 품질 이슈 발생률 | 통과 후 운영 중 품질 오류가 발생한 비율 | 품질 이슈 건수 / 운영 데이터 건수 |
| **C-3 계통** | 계통 커버리지 | 핵심 자산 중 계통이 구축된 비율 | 계통 연결 완료 자산 / 구축 대상 자산 |
| | 근거 추적 성공률 | AI 답변을 원천까지 되짚을 수 있는 비율 | 역추적 성공 건수 / 추적 시도 건수 |
| | 영향도 분석 수행률 | 변경 작업 전 영향도 분석을 수행한 비율 | 영향도 분석 수행 건수 / 변경 작업 건수 |

각 KPI의 "왜 필요한가·목표 설정"과 운영 성숙 시 넓히는 보조 지표는 해당 가이드 7장에 있다 — [C-1](../C-1%20Observability/C-1%20Observability.md#kpi) · [C-2](../C-2%20데이터%20품질%20관리/C-2%20데이터%20품질%20관리.md#kpi) · [C-3](../C-3%20데이터%20계통%20Lineage/C-3%20데이터%20계통%20Lineage.md#kpi).

<a id="s6"></a>

## 6. 다른 주제와의 관계

운영 기준은 신뢰성(C) 안에서 끝나지 않고 거버넌스(F)·메타데이터(A-2)와 이어진다. 이 문서는 "누가·무엇을·언제까지 책임지나"만 정하고, 실제 실행·정책은 아래 주제가 맡는다.

| 주제 | 운영 기준서와의 관계 (경계) |
|------|------|
| [F-1 데이터 운영관리](../F-1%20데이터%20운영관리/F-1%20데이터%20운영관리.md) | 운영 기준은 복구·변경의 책임(R&R)을 정하고, 멈춘 파이프라인 복구·재처리·변경 관리의 실행 절차(Runbook·롤백·변경 승인)는 F-1이 맡는다 |
| [F-2 데이터 생애주기 관리](../F-2%20데이터%20생애주기%20관리/F-2%20데이터%20생애주기%20관리.md) | 생애주기 운영 기준은 신뢰성 유지 관점만 정하고, 보존 기간 결정·폐기·아카이빙 정책은 F-2가 정본이다 |
| [F-4 AI 데이터 권한 보안](../F-4%20AI%20데이터%20권한%20보안/F-4%20AI%20데이터%20권한%20보안.md) | 데이터 오너가 승인하는 접근 권한·민감정보 통제(누가 볼 수 있나)의 실제 규칙은 F-4를 따른다 |
| [A-2 메타데이터](../A-2%20메타데이터/A-2%20메타데이터.md) | 정상 범위·품질 합격선·보존 기간 같은 기준값은 A-2 메타데이터에 기록해 같은 기준이 일관되게 적용되도록 한다 |
