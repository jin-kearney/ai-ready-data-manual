> 절대 원칙: 외부 시장통계 의존 금지 — 개념·구조 중심

## 1. 현업 Pain Point — 디지털화가 안 됐을 때 AI 활용에서 생기는 문제

### 1-1. 종이·수기 기록은 AI가 아예 읽지 못한다

종이에 쓴 검사 결과, 수기 정비 일지, 손으로 그린 스케치는 **검색도 집계도 불가능**하다. AI가 학습하거나 추론에 활용하려면 데이터가 컴퓨터가 읽을 수 있는 형식이어야 하는데, 종이 상태에서는 아예 입력 자체가 막힌다.

> **발전설비 현장 예시:** 터빈 정비 이력이 종이 작업지시서와 현장 엔지니어의 노트에 분산돼 있으면, AI 예측 유지보수 모델은 "과거에 이 부품이 몇 번 교체됐는가"조차 파악할 수 없다. 모델이 고장 패턴을 학습하려면 충분한 과거 이력 데이터가 필요하지만, 아날로그로 잠겨 있는 한 그 데이터는 존재하지 않는 것과 같다. [[1]](#ref1)

- 수기 검사표가 캐비닛에 쌓여 있어도 데이터로서 가치가 없음 — 검색 불가, 집계 불가, 모델 학습 불가
- 사진·스케치·현장 메모도 마찬가지 — 이미지 파일로 저장되지 않으면 AI가 볼 수 없음
- 검사 결과가 "특정 시점의 스냅샷"에 그침 — 시계열 분석·추세 파악이 불가능 [[2]](#ref2)

### 1-2. 한 번 입력한 데이터를 다시 입력하는 중복 작업

종이에 기록한 수치를 ERP·MES 시스템에 다시 입력하는 이중 작업이 발생한다. 이 과정에서:

- **입력 오류 발생** — 수기 기록 → 전산 입력 과정에서 작업자 판독 오류, 전사 실수
- **지연(lag) 발생** — 데이터가 기록되고 수집·집계되기까지 시간이 걸려 의사결정이 늦어짐 [[2]](#ref2)
- 종이 1만 장/인/년 소비, 보관 공간·분실 위험, 양식 업데이트 반영 지연 [[3]](#ref3)

### 1-3. 베테랑 은퇴로 암묵지(Tacit Knowledge)가 영원히 소실된다

이것이 가장 되돌리기 어려운 손실이다.

**암묵지의 구성:**
| 지식 유형 | 비중 | 내용 예시 |
|---|---|---|
| 명시지(Explicit) | ~10% | 절차서·매뉴얼 — 문서화 가능 |
| 묵시지(Implicit) | ~30% | 반복 경험으로 체득한 최적 방법 — 말로 설명 가능하나 문서화 안 됨 |
| 암묵지(Tacit) | ~60% | 감각·직관·판단 — "이 소리가 나면 베어링 마모", 현장 특유의 진동 해석 등 |

**소실 위험의 심각성:**
- 숙련 작업자 은퇴 시 그가 보유한 노하우의 약 **90%가 함께 사라짐** [[4]](#ref4)
- 제조업 전체로 보면 **핵심 운영 지식의 70%가 비문서화** 상태 [[5]](#ref5)
- 25세 이상 인력의 25%가 55세 이상 — 은퇴 파고가 이미 시작됨 [[5]](#ref5)

> **극단적 사례(일반 산업):** NASA 아폴로 새턴V 로켓 설계도는 남아 있었지만 실제 제작 방법을 아는 엔지니어가 모두 떠나 재현에 실패했다. 핵탄두 부품의 경우 은퇴 엔지니어의 암묵지 소실로 복원에 6,900만 달러·5년이 소요됐다 [[5]](#ref5). 발전설비 현장에서도 같은 구조의 위험이 존재한다 — 특정 터빈 기종의 조정·트러블슈팅 요령이 엔지니어 1~2명 머릿속에만 있을 때, 그 퇴직과 함께 기관 기억(institutional memory)이 사라진다.

**AI 관점의 문제:** 암묵지는 AI 학습 데이터로 변환되지 않으면 모델이 활용할 수 없다. "30년 베테랑이 이 경보를 무시하는 것은 ○○ 때문"이라는 판단 기준이 데이터로 남지 않으면 AI는 오경보만 계속 낼 것이다. [[6]](#ref6)

### 1-4. 종이 문서는 검색·분석이 불가능하다 = "다크 데이터(Dark Data)"

디지털로 수집됐지만 구조화되지 않거나, 수집조차 안 된 데이터를 **다크 데이터(Dark Data)**라 한다. 제조 현장에서 다크 데이터의 대표적 형태:

- 종이 검사표, 수기 작업지시서, 현장 스케치
- 아날로그 게이지 판독값(사진으로도 남기지 않은 것)
- 구두로만 인계된 교대 인수인계 사항
- 활용되지 않은 센서 로그(저장됐지만 분석 안 됨)

Gartner 정의 기준: 조직이 수집·저장하지만 다른 목적(분석·의사결정)에는 활용하지 못하는 정보 자산. [[7]](#ref7)

**다크 데이터의 비용:** 저장 비용은 발생하지만 가치는 창출하지 못한다. 의사결정·AI 모델·품질분석에서 이 데이터가 빠지면 판단의 빈틈이 생긴다. [[8]](#ref8)

## 2. 기대 효과 — "적용 전/후" 대비

### 2-1. Worked Example: 터빈 정기점검 기록지

| 구분 | 전(Before, 종이) | 후(After, 디지털화) |
|---|---|---|
| 기록 방식 | 현장 엔지니어 수기 기입, 캐비닛 보관 | 태블릿·현장 단말 입력, DB 자동 저장 |
| 검색 | 기호문 캐비닛 뒤져야 함 (30분~수시간) | 설비코드·날짜·항목으로 즉시 검색 |
| 집계·분석 | 수작업 엑셀 집계 (1~2일) | 자동 집계, 트렌드 그래프 즉시 |
| AI 활용 | 불가 — 종이는 모델이 읽지 못함 | 예측 유지보수 모델 학습 데이터로 투입 |
| 오류 위험 | 판독 오류·전사 오류 상시 | 입력 단계 단일화, 검증 로직 적용 |
| 인수인계 | 구두·메모 의존, 누락 발생 | 전 이력 DB로 남아 어느 직원이든 참조 |

> 발전소 디지털 유지보수 전환 사례: 디지털 CMMS 플랫폼 도입 후 **비계획 정지 35% 감소, 유지보수 인력 비용 28% 절감**. [[9]](#ref9)

### 2-2. Worked Example: 수기 품질 검사표

| 구분 | 전 | 후 |
|---|---|---|
| 불량 데이터 분류 | 작업자 주관 판단, 오분류 다발 | 표준 코드 선택식, 분류 일관성 확보 |
| 문제 보고 지연 | 유압 누유 — 발생 3시간 후 보고, $2,000 손실 [[10]](#ref10) | 현장 즉시 입력 → 자동 알림 발송 |
| 반복 불량 파악 | 인지 어려움 (캐비닛별 분산 이력) | 패턴 분석으로 뿌리원인 식별 |
| AI 품질검사 기초 | 라벨 없음, 학습 불가 | 분류 데이터 → AI 검사 모델 학습 소재 |

### 2-3. Worked Example: 베테랑 정비 노하우 디지털화

- **전:** 30년 경력 터빈 엔지니어의 "이 기종은 기동 시 ○○ 수치가 ○○ 이상이면 ○○ 부위 점검"이라는 판단이 머릿속에만 존재
- **후:** 비정형 인터뷰·작업 관찰·음성 녹취 등으로 규칙 추출 → 구조화된 지식 DB 구축 → AI 진단 보조 모델 기초 데이터로 활용
- **효과:** 문제 해결 속도 **15~25% 향상**, 품질 이슈 **8~12% 감소**, 신입 교육 기간 **20~30% 단축** [[4]](#ref4)

## 3. 적용 판단 — 무엇부터 디지털화할 것인가

### 3-1. 핵심 판단 기준 (5가지)

모든 아날로그 자료를 다 디지털화하는 것은 비효율이다. 아래 기준으로 우선순위를 정한다:

| 판단 기준 | 설명 | 제조 현장 예시 |
|---|---|---|
| **소실 위험** | 담당자 은퇴·문서 노후화·물리적 훼손 위험 | 특정 설비 기종만 아는 베테랑 엔지니어, 누락 직전 도면 |
| **AI 활용 가능성** | 디지털화하면 학습·검색·분석에 쓸 수 있는가 | 정기점검 이력 → 예측 유지보수 모델 학습 소재 |
| **업무 중요도** | 안전·품질·법규 준수에 직결되는가 | 안전밸브 점검표, 법정 의무 검사 기록 |
| **반복 활용성** | 한 번 디지털화하면 자주 쓰이는가 | 표준 작업지시서, 자주 참조하는 설비 이력 |
| **현장 의존도** | 특정 장소·사람 없이는 접근 불가한가 | 현장 캐비닛에만 있는 도면, 특정 담당자만 아는 절차 |

### 3-2. 우선순위 매트릭스 (가치 × 소실위험)

```
높음  │ [2순위]         │ [1순위] ◀ 최우선
소    │ 활용가치 높음   │ 활용가치 높음
실    │ 소실위험 낮음   │ 소실위험 높음
위    │ (여유 있게 추진)│ (즉시 착수)
험    ├────────────────┼─────────────────
낮음  │ [4순위]         │ [3순위]
     │ 활용가치 낮음   │ 활용가치 낮음
     │ 소실위험 낮음   │ 소실위험 높음
     │ (보류 또는 생략)│ (단순 보존만)
     └────────────────┴─────────────────
          낮음               높음
                 활용 가치
```

**1순위(즉시 착수):** 은퇴 임박 베테랑의 노하우, 오래된 설비 도면, 법정 보관 의무 기록
**2순위:** 자주 쓰이는 표준 절차서, 현장 검사 양식
**3순위:** 활용도 낮고 소실 위험 있는 기록 — 단순 보존
**4순위(보류·생략):** 활용 가능성도 낮고 소실 위험도 낮은 자료 [[11]](#ref11)

### 3-3. "굳이 안 해도 되는" 기준

- 이미 다른 시스템에 디지털로 존재하는 데이터의 단순 중복
- AI·분석 활용 계획이 없고, 반복 참조도 드문 과거 자료
- 보관 의무 기간이 지난 문서 (법규 확인 후)
- 디지털화 비용이 예상 활용 가치를 크게 초과하는 자료

## 4. 작게 시작 — 파일럿 접근

### 4-1. 단계별 접근법

1. **한 업무·한 양식 선정** — 가장 Pain이 크거나 AI 활용 연결이 명확한 단일 양식 하나
2. **병행 운영(2~4주)** — 종이와 디지털 동시 운영하며 오류·누락 점검. 디지털로 전환이 확인될 때까지 종이 백업 유지 [[12]](#ref12)
3. **효과 측정** — "입력 오류 ○% 감소", "검색 시간 ○분 단축" 등 구체적 수치로 확인
4. **인접 영역으로 확대** — 첫 양식의 성공 패턴을 유사 업무에 적용

> 예시: 터빈 정기점검 체크시트 한 종류 → 디지털 입력 전환 → 1개월 후 집계·검색 효과 확인 → 전체 정기점검 기록으로 확대 → 예측 유지보수 AI 모델 학습 데이터 파이프라인 연결

### 4-2. 시작점 선정 가이드

| 질문 | 예(Yes) | 아니오(No) |
|---|---|---|
| 이 자료가 없어지면 업무가 멈추거나 AI 계획에 차질이 생기는가? | 파일럿 후보 | 후순위 |
| 현재 이 자료를 찾는 데 10분 이상 걸리는가? | 파일럿 후보 | 검토 |
| 담당자가 1~2명에게 집중돼 있는가? | 파일럿 후보 | 검토 |
| 디지털화 후 AI·분석에 즉시 쓸 수 있는가? | 파일럿 후보 | 후순위 |

## 5. 다크 데이터(Dark Data) 일반론 — AI 관점의 의사결정 공백

- 기업 저장 데이터의 약 90%가 분석·의사결정에 활용되지 않는 다크 데이터로 추정됨 [[7]](#ref7)
- 이 데이터가 AI 학습·분석에서 빠지면 모델은 "보이는 것만"으로 판단하게 된다 — 제조 현장의 경우 가장 중요한 이력·경험·판단 기준이 종이 안에 잠겨 있는 상황
- 단순 저장 비용보다 "빠진 인사이트"의 비용이 더 크다 — 예측 실패, 반복 불량, 교육 비효율

**아날로그 다크 데이터의 특수성:** 디지털 다크 데이터는 적어도 컴퓨터가 읽을 수 있는 형식으로 존재한다. 아날로그 기록은 그 이전 단계 — **디지털화 없이는 처리 대상에도 들어오지 못한다.**

## 핵심 Take 5줄

1. **종이·수기 기록은 AI 입력 자체가 불가** — 아무리 방대한 현장 이력도 아날로그 상태에서는 AI 학습 소재가 되지 못한다.
2. **베테랑 은퇴 = 지식 소실** — 숙련 엔지니어 퇴직 시 노하우의 90%가 사라진다. 이는 일회성 손실이며 되돌릴 수 없다.
3. **전부 다 디지털화하는 게 아니다** — 소실위험 × 활용가치 매트릭스로 골라서 한다. 4순위(둘 다 낮음)는 안 해도 된다.
4. **발전·플랜트 현장은 "기록의 보고"** — 수십 년치 정비 이력, 점검표, 이상 대응 기록이 종이로 잠겨 있으면 예측 유지보수·고장 패턴 분석 AI는 출발도 못 한다.
5. **작게 시작해야 성공한다** — 한 양식 하나부터 병행 운영으로 검증하고 확대한다. 전사 일괄 전환은 실패율이 높다.

## 불확실 / 검증 필요 사항

- "수기 검사표 종이 → AI 학습 데이터"로 전환한 두산에너빌리티 또는 동종 대형 발전사의 구체적 수치(소요 기간, 전환 비율, AI 정확도 개선폭) — 실제 발표 본편에서 as-is 수치를 채워야 함
- 암묵지 디지털화 도구(음성 인터뷰, 작업 관찰 AI 등)의 한국 제조업 적용 사례 — 추가 리서치 가능
- 매트릭스의 판단 기준 가중치(소실위험 vs 활용가치 중 어느 쪽을 더 중시할지)는 기업별 상황에 따라 달라짐 — 본편에서 "상황에 따라 조정" 취지로 기술 권장
- 발전설비 법정 기록물의 보존 의무 기간(산업안전보건법·전기사업법 기준) — 규제 근거 별도 확인 필요
