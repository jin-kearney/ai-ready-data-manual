# F-3 데이터 디지털화 — 작업 메모

이 주제 전용 규칙·결정·피드백 누적 메모. 가이드 수정 전 먼저 읽는다.

## 정본 모델 / 핵심 결정 (2026-06-29 초안)

- **주제 성격: 선택형** — Why와 When을 한 섹션으로 통합(1장 "Why — 왜, 언제 필요한가"). B-3 온톨로지와 같은 형식. **KQ 가시화 박스는 두지 않는다**(B-3 방식).
- **섹션 구조(5섹션 + 별첨):** 1 Why(+When) → 2 What → 3 How → 4 Tech Stack → 5 Where. KPI 없음(C군 아님), 로드맵 없음. Where가 마지막.
- **MECE 경계(엄수):** F-3 = 아날로그→디지털 **일회성·대량 오프라인 전환**만. 이미 디지털인 문서 정제·구조화는 [[B-1 전처리]], 품질 판정은 [[C-2]], 자산 등록·검색은 [[A-1]], 메타 스키마는 [[A-2]], 보존·폐기는 [[F-2]]. 전환 절차 안의 "메타데이터 태깅"은 실행만 F-3, 스키마 정의는 A-2.
- **계열사 예시 = 두산에너빌리티**(발전설비·터빈·대형 플랜트). 모든 구체값은 가상(상단 예시 표기 안내).
- **데이터 준비 관점 고정:** OCR/STT 모델 만드는 법이 아니라 아날로그 자산을 디지털 데이터 자산으로 전환·정비하는 법.

## 정본 용어 (끝까지 일관)
- 전환 방식: 스캔 · OCR(인쇄 글자 인식) · ICR(수기 글자 인식) · STT(음성→텍스트) · 모바일/태블릿 입력 · 인터뷰 구조화.
- 검수: **신뢰도(Confidence Score)** 구간(95%↑ 자동 / 80~95% 샘플링 / 80%↓ 전량 검수). 수치는 PoC 캘리브레이션.
- 지속 수집: Shadow Data(관리 밖 산재) → 공식 자산 승급 5단계. Digital-first = 처음부터 디지털 입력(e-form).
- 전환 절차 5단계: 조사 → 우선순위 → 전환 → 검수 → 자산 등록.
