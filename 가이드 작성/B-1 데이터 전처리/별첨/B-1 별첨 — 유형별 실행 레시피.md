# B-1 별첨 — 유형별 전처리 실행 레시피

이 문서는 [B-1 데이터 전처리 가이드](../B-1%20데이터%20전처리.md)의 실행 보조 자료다. 본문 5장(어떻게 변환·운영하나)에서 다룬 유형별 전처리 방식을 엔지니어가 그대로 따라 할 수 있도록 코드·프롬프트·완성 사례로 펼쳤다. 본문이 "무엇을 왜"를 설명한다면, 이 문서는 "코드로 어떻게"를 보여준다.

- 3-D 유형별 실행 레시피: 8개 문서 유형의 코드와 LLM 프롬프트
- 3-E 유형별 worked example: 가상 원본에서 전처리를 거쳐 결과까지
- 3-F 한 양식 완성 사례: 스키마 먼저, 값은 그다음으로 한 양식을 끝까지

모든 예시는 두산 제조 맥락의 가상 데이터이며 실데이터가 아니다. 코드의 라이브러리 API는 버전에 따라 달라질 수 있으므로 적용 시 공식 문서로 확인한다.

### [Backup 3-D] 유형별 전처리 실행 레시피 (따라하기)

문서 유형마다 **그대로 따라 할 수 있는** 최소 코드와 LLM 프롬프트를 모았다. 각 레시피는 `준비물 → 코드(또는 LLM) → 입력 예시 → 출력 예시` 순서다.

> **읽는 법 / 원칙 3가지.**
> 1. 코드는 **가장 표준적인 오픈소스 라이브러리**로 최소 골격만 보였다(실제로는 예외 처리·로깅을 더한다). 파일명·시트명·페이지는 예시값이니 바꿔 쓴다.
> 2. **언제 코드, 언제 LLM?** — 양식이 일정하고 규칙이 명확하면 **코드**(빠르고 싸고 결정적), 양식이 제각각이거나 병합·계층·조건부라 코드로 짜기 어려우면 **LLM에 "스키마 먼저, 값은 그다음"으로** 시킨다(→ 이 방법론이 [5.2](../B-1%20데이터%20전처리.md#sec62) 핵심 실행법). 둘을 섞는 게 보통이다(코드로 뽑고 LLM으로 정돈).
> 3. **여기서 LLM은 "데이터를 정돈하는 파서·정리 도구"로 쓰는 것**이지, 챗봇·RAG 앱을 만드는 게 아니다([데이터 준비 관점](../B-1%20데이터%20전처리.md#s11) 유지). LLM 출력은 **항상 JSON 스키마로 강제**하고, **원문 값과 대조 검증**하며, **LLM이 값을 지어내지 않게**("원문에 없으면 null") 지시한다.

모든 레시피의 출력은 [2.3](../B-1%20데이터%20전처리.md#kq3)의 공통 형태 — `chunk_id · type · content · provenance(문서ID·위치)` — 를 목표로 한다.

#### 레시피 1 — PDF 표 추출 (코드: pdfplumber / Camelot)

**준비물:** `pip install pdfplumber` (또는 표 전용 `pip install "camelot-py[base]"`)

**실전 함정 (그냥 `extract_tables()`만 하면 깨지는 곳):**

| 함정 | 증상 | 대응 |
|---|---|---|
| 스캔 PDF | 텍스트 레이어가 없어 빈 결과 | 먼저 판별 → OCR 경로(레시피 4 하단) |
| 테두리 없는 표 | 격자 인식 실패, 열이 합쳐짐 | `vertical_strategy="text"`(공백 정렬) |
| 셀 안 줄바꿈 | 한 셀이 여러 줄로 쪼개짐 | 셀 값의 `\n` 정리 |
| 표가 페이지를 넘김 | 페이지마다 별도 표로 끊김 | 같은 머리글이면 이어 붙이고 **반복 머리글 제거** |
| 다단(2열) 레이아웃 | 좌·우단이 한 줄로 섞임 | 표 영역만 crop하거나 Camelot `table_areas` 지정 |

```python
import pdfplumber

def is_scanned(page):                       # 텍스트 레이어 없으면 스캔본 → OCR 경로로
    return not (page.extract_text() or "").strip()

# 테두리 없는 표는 세로를 '공백 정렬', 가로는 '선'으로 잡는다(양식 따라 조정)
TABLE_SETTINGS = {"vertical_strategy": "text", "horizontal_strategy": "lines"}

header, all_rows = None, []
with pdfplumber.open("QR-2024-0123.pdf") as pdf:
    for pno, page in enumerate(pdf.pages[44:47], start=45):   # 표가 45~47p로 이어진다고 가정
        if is_scanned(page):
            continue                                          # OCR(레시피 4)로 우회
        for t_no, table in enumerate(page.extract_tables(TABLE_SETTINGS), start=1):
            # 셀 안 줄바꿈·공백 정리
            table = [[(c or "").replace("\n", " ").strip() for c in row] for row in table]
            if header is None:
                header, body = table[0], table[1:]
            else:                                             # 이어지는 페이지: 머리글 반복이면 제거
                body = table[1:] if table[0] == header else table
            for r_no, row in enumerate(body, start=1):
                all_rows.append({
                    "chunk_id": f"QR-2024-0123_p{pno}_tbl{t_no}_r{r_no:02d}",
                    "type": "table_row",
                    "content": dict(zip(header, row)),
                    "provenance": {"doc_id": "QR-2024-0123", "page": pno,
                                   "table_no": t_no, "row": r_no},
                })

# 검증 — 모든 행의 열 수가 머리글과 같은지(어긋나면 표 인식이 깨진 것)
bad = [c["chunk_id"] for c in all_rows if len(c["content"]) != len(header)]
assert not bad, f"열 수 불일치(표 인식 의심): {bad}"
```

> 테두리 없는 표·고밀도 수치표는 [Camelot](https://camelot-py.readthedocs.io)이 더 정확하다: `camelot.read_pdf("f.pdf", pages="45", flavor="lattice")`(선 격자) 또는 `flavor="stream"`(공백) → `tables[0].df`(DataFrame), `tables[0].parsing_report["accuracy"]`로 신뢰도 확인. 표 위치가 일정하면 `table_areas=["x1,y1,x2,y2"]`로 영역을 고정하면 다단 혼입을 막는다.

**LLM 보강(규칙으로 안 떨어질 때) — 프롬프트 템플릿:**
```text
너는 문서 표 정돈기다. 아래는 PDF에서 추출됐지만 행/열이 뒤섞인 표 텍스트다.
원래의 행/열 구조를 복원해 JSON 배열로만 출력하라.
규칙: (1) 원문에 있는 값만 쓴다 — 없는 값은 지어내지 말고 null.
(2) 숫자·단위는 원문 그대로(반올림·환산 금지). (3) 행 수는 원문과 같게.
(4) 설명·말머리 없이 JSON 배열만.
[표 텍스트]
용접부 외관 0.05mm 0.03mm 합격 도장 두께 80㎛ 76㎛ 합격
```
→ 출력: `[{"공정":"용접부","검사항목":"외관","합격기준":"0.05mm","실측값":"0.03mm","판정":"합격"}, …]`
> LLM 출력은 반드시 **행 수·핵심 수치를 원문과 대조**한 뒤 통과시킨다(환각 방지).

#### 레시피 2 — Excel: 머리글 탐지·병합·색·소계까지 (코드: openpyxl / pandas)

**준비물:** `pip install openpyxl pandas`

"병합 풀기"는 시작일 뿐이다. 실무의 점검표·집계표는 아래가 한꺼번에 섞여 있다.

| 함정 | 왜 깨지나 | 대응 |
|---|---|---|
| **한 시트에 여러 표·제목·요약 행** | 1행을 머리글로 가정하면 어긋남 | 실제 데이터 표의 머리글 행을 탐지 |
| **다중 행 머리글**(상위 카테고리+하위 항목) | 머리글이 2~3행에 걸침 | 행을 합쳐 `카테고리.항목` 한 줄 컬럼명으로 |
| **머리글 병합 ↔ 데이터 병합** | 채우는 **방향이 다름**(가로 vs 세로) | 머리글은 가로 복제, 데이터는 세로 forward-fill |
| **수식 캐시값 없음** | `data_only`인데 `None` | Excel로 한 번 저장하거나 수식 평가 |
| **색·아이콘에만 담긴 판정** | 빨강=이상이 값으로 안 남음 | 직접 채운 색은 `fill`, 조건부서식은 **규칙 평가** 필요 |
| **소계/합계/단위 행** | 관측이 아닌 행이 섞임 | 키워드·빈 행으로 제거 |
| **숫자에 섞인 단위·콤마·`-`** | `"80㎛"`·`"1,200"`·`"N/A"` | 숫자만 추출, 단위는 별도 컬럼 |

```python
import openpyxl, pandas as pd, re

wb = openpyxl.load_workbook("설비점검표.xlsx", data_only=True)   # 수식의 '결과값'을 읽음
ws = wb["점검"]

# 0) 병합 처리 — 머리글 영역은 가로 복제, 데이터 영역은 세로 채움
HEADER_ROWS = 2                                  # 머리글이 2행(상위 카테고리 + 하위 항목)
for rng in list(ws.merged_cells.ranges):
    v = ws.cell(rng.min_row, rng.min_col).value
    ws.unmerge_cells(str(rng))
    for row in ws[str(rng)]:
        for cell in row:
            cell.value = v                       # 병합 영역 전체에 좌상단 값 복제

# 1) 다중 행 머리글 → 한 줄 컬럼명으로 평탄화
rows = list(ws.values)
columns = []
for col in zip(*rows[:HEADER_ROWS]):             # 머리글 2행을 세로로 묶어
    parts = [str(x).strip() for x in col if x not in (None, "")]
    columns.append(".".join(dict.fromkeys(parts)) or "col")   # '윤활.결과' 식
df = pd.DataFrame(rows[HEADER_ROWS:], columns=columns)

# 2) 소계/합계/빈 행 제거
df = df.dropna(how="all")
df = df[~df.iloc[:, 0].astype(str).str.contains("합계|소계|총계|평균|단위", na=False)]

# 3) 숫자 정제: "1,200"·"80㎛"·"-"·"N/A" → 숫자 / 단위는 분리
def to_num(x):
    s = str(x).replace(",", "").strip()
    if s in ("", "-", "N/A", "NA", "None"):
        return None
    m = re.match(r"^-?\d+(\.\d+)?", s)
    return float(m.group()) if m else None

# 4) 색에만 담긴 '이상' 판정 데이터화 (직접 채운 셀 색일 때)
def is_red(cell):
    rgb = getattr(cell.fill.fgColor, "rgb", None)
    return isinstance(rgb, str) and rgb.endswith("FF0000")    # 예: 'FFFF0000'

# 5) 가로 교차표(wide) → 레코드(long): "한 행 = 한 관측"
long = df.melt(id_vars=["설비", "카테고리"], value_vars=["1주차", "2주차", "3주차"],
               var_name="점검주차", value_name="결과")
long["이상여부"] = long["결과"].astype(str).eq("이상")

# 6) 검증 — 관측 수 = (원본 데이터 행 수 × 펼친 주차 수)인지 대조(누락·중복 탐지)
assert len(long) == len(df) * 3, "정규화 과정에서 행 수가 어긋남"
print(long.to_dict("records"))
```

> **수식·색 주의:** ① `data_only=True`는 **Excel이 저장하며 캐시한 결과값**을 읽는다 — 엑셀이 아닌 도구로 만든 파일은 캐시가 없어 `None`이 나올 수 있다(한 번 열어 저장하거나 [`formulas`](https://pypi.org/project/formulas/)로 평가). ② **조건부서식**으로 칠해진 색은 셀에 저장되지 않고 "규칙"으로만 있어 `cell.fill`로 안 읽힌다 — `ws.conditional_formatting` 규칙을 직접 평가하거나, 애초에 색이 아니라 값으로 판정을 받도록 양식을 바꾸는 게 낫다.

→ 출력: `[{"설비":"CNC-01","카테고리":"윤활","점검주차":"3주차","결과":"이상","이상여부":true}, …]`

#### 레시피 3 — 이메일: 헤더·HTML·인용·서명·첨부·스레드 (코드 + LLM)

**준비물:** 표준 라이브러리 `email` + `pip install talon html2text`

**실전 함정:**

| 함정 | 대응 |
|---|---|
| 인코딩된 헤더 `=?UTF-8?B?…?=` (한글 제목·이름) | `policy.default`가 자동 디코드 |
| HTML 전용 메일(평문 없음) | HTML→평문 변환(표·목록 보존) |
| 답장이 위/아래/원문 사이(top·bottom·inline) | `talon`이 위치 무관하게 인용 제거 |
| 서명·면책문이 본문에 붙음 | 서명 분리기로 떼어 별도 보관 |
| 스레드 중복(같은 내용 N번) | `In-Reply-To`/`References`로 묶고 최신만 |
| 본문 내장 이미지 vs 진짜 첨부 | `Content-ID` 유무로 구분 |

```python
from email import policy
from email.parser import BytesParser
from talon import quotations
from talon.signature.bruteforce import extract_signature   # 모델 불필요(규칙 기반)
import html2text

with open("mail.eml", "rb") as f:
    msg = BytesParser(policy=policy.default).parse(f)        # 인코딩 헤더 자동 디코드

# 1) 헤더 → 메타데이터 + 스레드 키
meta = {"message_id": msg["Message-ID"], "in_reply_to": msg["In-Reply-To"],
        "references": msg["References"], "from": msg["From"], "to": msg["To"],
        "date": msg["Date"], "subject": msg["Subject"]}

# 2) 본문: 평문 우선, 없으면 HTML→평문(표·목록 유지)
plain = msg.get_body(preferencelist=("plain",))
if plain:
    text = plain.get_content()
else:
    h = msg.get_body(preferencelist=("html",))
    conv = html2text.HTML2Text(); conv.body_width = 0        # 줄바꿈 강제 끔
    text = conv.handle(h.get_content()) if h else ""

# 3) 인용된 과거 메일 제거 → 서명 분리 → '이번 메일 새 내용'만
reply = quotations.extract_from(text, "text/plain")
body, signature = extract_signature(reply)                   # (본문, 서명) 분리

# 4) 첨부: 본문 내장 이미지(Content-ID)와 진짜 첨부 구분
attachments, inline = [], []
for part in msg.iter_attachments():
    item = {"filename": part.get_filename(), "ctype": part.get_content_type()}
    (inline if part.get("Content-ID") else attachments).append(item)
# → 진짜 첨부(PDF·Excel)는 레시피 1·2로 보내고, 본문과 message_id로 연결
```

> 규칙으로 안 떨어지는 인라인 답변·한글 서명은 **LLM에게 나눠 달라고** 하는 편이 안정적이다:

```text
너는 이메일 정리기다. 아래 본문을 네 부분으로 나눠 JSON으로만 출력하라.
키: latest_reply(이번에 새로 쓴 내용), quoted(인용된 과거 메일),
    signature(서명), disclaimer(법적 면책문).
규칙: 원문 문장을 그대로 '분류만' 한다(요약·번역·변형 금지). 해당 없으면 "".
설명 없이 JSON만.
[본문]
…(메일 본문 붙여넣기)…
```

→ 출력 청크: `{"type":"email","content":"<body>","provenance":{"doc_id":"<Message-ID>","thread":"<References>","from":"…","date":"…"},"attachments":[…]}`

#### 레시피 4 — Word·HWP: 순서 보존·제목 계층·표 병합 (코드)

**준비물:** Word는 `pip install python-docx`(또는 의미 보존이 좋은 `pip install mammoth`)

**실전 함정:** ① `doc.paragraphs`와 `doc.tables`를 따로 읽으면 **본문 속 표의 끼인 위치가 사라진다** → 문서 순서대로 순회해야 함. ② 제목이 큰 글씨가 아니라 **스타일**(Heading 1/2)로만 구분됨. ③ docx 표의 **병합셀은 같은 텍스트를 반복** 반환. ④ 변경내용 추적(`w:ins`/`w:del`)이 본문에 섞임.

```python
from docx import Document
from docx.table import Table
from docx.text.paragraph import Paragraph

def iter_blocks(doc):                          # 문단·표를 '문서에 나온 순서대로' 순회
    for child in doc.element.body.iterchildren():
        if child.tag.endswith("}p"):
            yield Paragraph(child, doc)
        elif child.tag.endswith("}tbl"):
            yield Table(child, doc)

doc, blocks = Document("보고서.docx"), []
for blk in iter_blocks(doc):
    if isinstance(blk, Paragraph):
        if not blk.text.strip():
            continue
        st = blk.style.name or ""
        if st.startswith("Heading"):           # 스타일 → Markdown 제목 계층
            lvl = int("".join(filter(str.isdigit, st)) or "1")
            blocks.append("#" * lvl + " " + blk.text)
        else:
            blocks.append(blk.text)
    else:                                       # Table: 병합셀 중복 텍스트 정리
        for row in blk.rows:
            seen, prev = [], object()
            for c in row.cells:
                seen.append("" if c.text == prev else c.text)   # 병합 반복분은 공란
                prev = c.text
            blocks.append(" | ".join(seen))
```

> 위치까지 한 번에 살리려면 `mammoth.convert_to_markdown(open("보고서.docx","rb")).value`가 편하다(제목·표·목록을 Markdown으로). **변경내용 추적·메모**가 있으면 python-docx로 깔끔히 못 떼니, Word/LibreOffice에서 "변경 내용 적용" 후 저장하거나 XML(`w:ins`/`w:del`)을 직접 처리한다.

#### 레시피 5 — CSV·로그·센서: 인코딩·머리말·단위·타임스탬프·대용량 (코드)

**준비물:** `pip install pandas charset-normalizer`

**실전 함정:**

| 함정 | 증상 | 대응 |
|---|---|---|
| 인코딩(cp949/euc-kr/BOM) | 한글 깨짐 `ì„¤ë¹„` | 자동 감지 후 명시 |
| 구분자 혼재(`,` `\t` `;`) | 한 열로 뭉침 | `sep=None`+`engine="python"` 추론 |
| 머리글 위 장비정보 머리말 | 1행을 머리글로 오인 | 머리글 행 탐지 후 `skiprows` |
| 머리글 아래 **단위 행**(`[℃]`) | 숫자 열이 문자열化 | 단위 행 제거 |
| 숫자에 콤마·단위·`ERR` | `to_numeric` 실패 | 숫자만 추출 |
| 타임스탬프 형식 혼재·중복·역순 | 정렬·집계 오류 | 파싱→중복제거→정렬 |
| 초대용량(수천만 행) | 메모리 초과 | `chunksize`로 청크 읽기 |

```python
import pandas as pd
from charset_normalizer import from_path

# 1) 인코딩 자동 감지
best = from_path("sensor_log.csv").best()
enc = best.encoding if best else "utf-8"

# 2) 머리글 행 탐지(위에 장비정보 머리말이 N줄 있을 수 있음)
def find_header(path, enc, key="timestamp"):
    with open(path, encoding=enc, errors="replace") as f:
        for i, line in enumerate(f):
            if key.lower() in line.lower():
                return i
    return 0

skip = find_header("sensor_log.csv", enc)
# 3) 일단 전부 문자열로 읽어 직접 형변환(자동 추론이 단위·콤마에 속지 않게)
df = pd.read_csv("sensor_log.csv", encoding=enc, sep=None, engine="python",
                 skiprows=skip, dtype=str, on_bad_lines="warn")

# 4) 머리글 바로 아래 '단위 행' 제거 ([℃],[bar] 등)
if df.iloc[0].astype(str).str.contains(r"\[|\]|단위", regex=True).any():
    df = df.iloc[1:].reset_index(drop=True)

# 5) 숫자 컬럼 정제: 콤마·단위·오류표기 → 숫자
for c in ["온도", "압력", "진동"]:
    df[c] = pd.to_numeric(df[c].str.replace(",", "", regex=False)
                                .str.extract(r"(-?\d+\.?\d*)")[0], errors="coerce")

# 6) 타임스탬프: 형식 혼재·중복·역순 정리
df["ts"] = pd.to_datetime(df["timestamp"], errors="coerce", utc=True)
df = (df.dropna(subset=["ts"]).drop_duplicates("ts")
        .sort_values("ts").set_index("ts"))

# 7) 대용량 다운샘플(1분 평균) + 짧은 결측만 보간
out = df[["온도", "압력", "진동"]].resample("1min").mean().interpolate(limit=5)

# (가로로 센서가 펼쳐진 표면 레시피 2처럼) df.melt(id_vars=["ts","설비"], var_name="센서", value_name="값")
# (초대용량은) for chunk in pd.read_csv(path, chunksize=100_000): ... 로 청크 처리
```

> 타임존·1ms 단위 초대용량은 **집계 정책(평균/최댓값/다운샘플 주기)을 먼저 합의**한다. 적재 방식(증분 vs 배치)은 [5.4](../B-1%20데이터%20전처리.md#sec63)·[Backup 6-B](../B-1%20데이터%20전처리.md#backup-6-b-배치-vs-증분-처리).

#### 레시피 6 — 채팅·회의록: 다양한 포맷·발화자 정규화·병합 (코드 + LLM, 경계 주의)

**준비물:** 표준 라이브러리 `re` (회의 자막이면 `pip install webvtt-py`)

**실전 함정:** ① 발화자·시각 표기가 도구마다 다름(카톡 PC·Teams·Zoom). ② 한 사람의 연속 발화가 여러 줄로 끊김. ③ 별칭/사번/실명이 섞임. ④ 시스템 메시지·이모지·첨부 알림 줄. ⑤ Zoom/Teams 자막은 `.vtt`/`.srt`.

```python
import re

# 여러 메신저/회의록 포맷을 동시에 시도(맞는 첫 패턴 사용)
PATTERNS = [
    re.compile(r"^\[(?P<time>\d{1,2}:\d{2}(:\d{2})?)\]\s*(?P<speaker>[^:]+):\s*(?P<text>.*)$"),
    re.compile(r"^(?P<speaker>.+?),\s*\[(?P<time>[^\]]+)\]\s*(?P<text>.*)$"),   # 카톡 PC 내보내기
    re.compile(r"^(?P<time>\d{2}:\d{2})\s+(?P<speaker>\S+)\s+(?P<text>.+)$"),    # Zoom/Teams
]
ALIAS = {"김검사원": "김검사", "kim.gs": "김검사"}        # A-3 Glossary 표준 인물로
SYSTEM = re.compile(r"(입장했습니다|나갔습니다|들어왔습니다|사진을 보냈|동영상|^파일:|이모티콘)")

utterances = []
for raw in open("회의록.txt", encoding="utf-8"):
    line = raw.rstrip("\n")
    if not line.strip() or SYSTEM.search(line):           # 빈 줄·시스템 줄 제거
        continue
    m = next((p.match(line) for p in PATTERNS if p.match(line)), None)
    if m:
        d = m.groupdict()
        d["speaker"] = ALIAS.get(d["speaker"].strip(), d["speaker"].strip())
        # 같은 사람 연속 발화면 병합
        if utterances and utterances[-1]["speaker"] == d["speaker"]:
            utterances[-1]["text"] += " " + d["text"]
        else:
            utterances.append({"time": d.get("time"), "speaker": d["speaker"],
                               "text": d["text"]})
    elif utterances:                                       # 패턴 없는 이어지는 줄
        utterances[-1]["text"] += " " + line.strip()
```

> 회의 자막(.vtt)은 `import webvtt; [(c.start, c.text) for c in webvtt.read("회의.vtt")]`로 (시각, 발화)을 뽑는다. 발화자 표기가 패턴으로 안 잡힐 만큼 자유로우면 LLM에 맡긴다 — 따라 할 프롬프트:

```text
너는 회의록 정리기다. 아래 대화를 발화 단위로 나눠 JSON 배열로만 출력하라.
키: time(시각, 없으면 null), speaker(발화자), text(발화 내용).
규칙: (1) 원문 문장 그대로 — 요약·번역·창작 금지. (2) 같은 사람의 연속 발화는 하나로 합쳐라.
(3) "입장했습니다" 등 시스템 줄은 제외. (4) 설명 없이 JSON 배열만.
[대화]
[10:03] 김검사: 로트 0312 표면 결함이요
재현됩니다
[10:04] 이엔지: RT 다시 돌려보죠
```
→ 출력: `[{"time":"10:03","speaker":"김검사","text":"로트 0312 표면 결함이요 재현됩니다"}, {"time":"10:04","speaker":"이엔지","text":"RT 다시 돌려보죠"}]`
>
> **경계:** 여기까지가 B-1(형식 변환)이다 — 발화자·시각·발화 단위로 **정돈**까지. "회의 결정사항·요약·액션아이템·감정 라벨"은 **의미를 입히는 일이라 [B-2 데이터 해설·주석](../../B-2%20데이터%20해설·주석/B-2%20데이터%20해설·주석.md)의 몫**이다 — B-1 레시피에서는 하지 않는다.

#### 레시피 7 — 이미지·스캔본: 보정 → OCR → 저신뢰 셀 검수 (코드 + LLM)

**준비물:** `pip install pillow pytesseract` + Tesseract 엔진(한국어 데이터 `kor` 포함 설치). 표 구조가 중요하면 클라우드 문서지능(Azure DI·Google Doc AI)이나 TSR 도구를 함께.

**실전 함정:**

| 함정 | 대응 |
|---|---|
| 기울기·그림자·저대비 | 그레이스케일·이진화·기울기 보정(deskew)으로 OCR 전 보정 |
| 글자 깨짐·오인식(0↔O, 1↔l) | 단어별 신뢰도(confidence)를 받아 낮은 것만 사람 검수 |
| 손글씨·도장·노이즈 | 정확도 낮음 → 검수 대상으로 분리(자동 신뢰 금지) |
| 표 구조 | OCR은 글자만 → 표는 TSR(표 구조 인식)·클라우드로 별도 |

```python
from PIL import Image
import pytesseract

img = Image.open("scan_검사일지.png").convert("L")    # 그레이스케일(전처리 1단계)
# (실무에선 여기서 OpenCV로 이진화·기울기 보정(deskew)을 더해 정확도를 올린다)

# 단어별 신뢰도까지 받기 (한국어+영어)
data = pytesseract.image_to_data(img, lang="kor+eng",
                                 output_type=pytesseract.Output.DICT)

words = []
for i, txt in enumerate(data["text"]):
    if not txt.strip():
        continue
    conf = int(data["conf"][i])
    words.append({"text": txt, "conf": conf,
                  "review": conf < 60})            # 신뢰도 60 미만은 사람 검수 플래그
low = [w["text"] for w in words if w["review"]]
print("검수 필요:", low)                            # 이 단어들만 사람이 확인
```

> OCR 텍스트가 깨끗하지 않을 때 **LLM으로 오타 교정·구조 복원**을 시킬 수 있다 — 단, 환각 위험이 크므로 규칙을 강하게:

```text
너는 OCR 결과 정리기다. 아래는 스캔 문서를 OCR한, 오타 섞인 텍스트다.
명백한 OCR 오인식(0↔O, 1↔l, 깨진 자모)만 교정하고 표 구조를 복원하라.
규칙: (1) 의미를 바꾸거나 없는 수치를 만들지 마라. (2) 판독 불가는 "[?]"로 표기.
(3) 설명 없이 결과 텍스트(또는 표)만.
[OCR 텍스트]
…(붙여넣기)…
```

> **경계:** 종이 문서를 **대량으로** 스캔·디지털화하는 일 자체는 [F-3 디지털화](../../F-3%20데이터%20디지털화/F-3%20데이터%20디지털화.md)의 몫이다. B-1은 그 결과(이미지)를 구조화하고, **저신뢰 셀은 자동 신뢰하지 않고 검수로 돌린다**.

#### 레시피 8 — PPT 슬라이드: 의미 단위로 묶기 (코드 + LLM)

**준비물:** `pip install python-pptx`

**실전 함정:** ① 텍스트박스가 화면 위치가 아니라 **만든 순서(Z-order)**로 저장돼 그냥 뽑으면 순서가 뒤죽박죽 → **좌표(위→아래, 좌→우)로 재정렬**. ② 제목·본문·표·도형·발표자 노트가 흩어짐 → **슬라이드 1장 = 1청크**로 묶기. ③ 차트는 글자가 아니라 내장 데이터 → 값을 표로 빼거나 캡션으로.

```python
from pptx import Presentation

prs = Presentation("기술자료.pptx")
chunks = []
for sno, slide in enumerate(prs.slides, start=1):
    title = slide.shapes.title.text if slide.shapes.title else ""
    bodies, tables = [], []
    # 좌표로 재정렬해 '사람이 보는 순서'로 (Z-order 뒤섞임 교정)
    for sh in sorted(slide.shapes, key=lambda s: (s.top or 0, s.left or 0)):
        if sh == slide.shapes.title:
            continue                                  # 제목은 위에서 이미 뽑음
        if sh.has_table:
            tables.append("\n".join("| " + " | ".join(c.text for c in row.cells) + " |"
                                    for row in sh.table.rows))
        elif sh.has_text_frame and sh.text.strip():
            bodies.append(sh.text.strip())
    notes = (slide.notes_slide.notes_text_frame.text
             if slide.has_notes_slide else "")
    chunks.append({"chunk_id": f"기술자료_p{sno}", "type": "slide", "title": title,
                   "body": bodies, "table": tables, "notes": notes,
                   "provenance": {"doc_id": "기술자료", "slide": sno}})
```

> 표·도형이 복잡하거나 차트 데이터까지 살려야 하면, 위에서 뽑은 **조각을 LLM에 주고 사람이 읽는 구조로 정리**시킨다 — 따라 할 프롬프트:

```text
너는 슬라이드 정리기다. 아래는 PPT 한 장에서 뽑은 제목·텍스트·표 조각인데 순서가 뒤섞여 있다.
사람이 읽는 순서로 정리해 JSON으로만 출력하라.
키: title(제목), body(불릿 텍스트 배열), table(있으면 Markdown 표), notes(발표자 노트).
규칙: (1) 원문 문구만 사용 — 요약·창작 금지. (2) 없으면 null 또는 빈 배열.
(3) 설명·말머리 없이 JSON만.
[슬라이드 조각]
제목: 터빈 블레이드 용접 결함 유형
크랙 표면 균열 / 기공 내부 공동 / 언더컷 모재 패임
표: 결함 허용 검출법 / 크랙 불허 PT / 기공 Ø0.5↓ RT
노트: 고온부는 PT 우선
```
→ 출력:
```json
{"title":"터빈 블레이드 용접 결함 유형",
 "body":["크랙 — 표면 균열","기공 — 내부 공동","언더컷 — 모재 패임"],
 "table":"| 결함 | 허용 | 검출법 |\n| 크랙 | 불허 | PT |\n| 기공 | Ø0.5↓ | RT |",
 "notes":"고온부는 PT 우선"}
```

#### ㉠ PDF 품질 검사 보고서 (표)

**① 원본(가상)** — 단순 추출 시 표가 한 줄로 뭉개진 모습:
```
품질검사성적서 QR-2026-0481  3페이지
```

**② 전처리** — *스키마 먼저:* `{공정, 검사항목, 기준, 실측, 판정}` 으로 확정 → LLM/표파서로 행 복원 → 문서ID·페이지를 provenance로 부착.
**③ 결과:**
```json
[
 {"chunk_id":"QR-2026-0481_p3_r1","공정":"용접부","검사항목":"외관","기준":"균열 없음","실측":"균열 없음","판정":"합격",
```

#### ㉡ PPT 기술 교육자료 (슬라이드)

**① 원본(가상)** — 슬라이드 7장: 제목 1개, 불릿 3개, 작은 표 1개, 발표자 노트.
**② 전처리** — 슬라이드 **1장 = 1청크**로 묶고 제목·본문·표·노트를 분류, 슬라이드 번호를 위치로 부착.
**③ 결과:**
```json

 "table":"| 결함 | 허용 | 검출법 |\n| 크랙 | 불허 | PT |\n| 기공 | Ø0.5↓ | RT |",
 "notes":"고온부는 PT 우선","provenance":{"doc_id":"TECH-블레이드","slide":7}}
```
→ *무엇이 좋아졌나:* 슬라이드 흩어진 정보가 의미 단위로 묶여, "언더컷 검출법" 질문에 슬라이드 7을 근거로 답한다.

#### ㉢ Excel 설비 점검표 (병합 + 가로 교차표)

**① 원본(가상)** — '윤활' 카테고리가 병합되고, 주차가 가로로 펼쳐지고, 이상은 빨강 색으로만 표시:
```
설비    | 카테고리 | 1주차 | 2주차 | 3주차
CNC-01 | 윤활(병합)| 정상  | 정상  | [빨강]이상
```
**② 전처리** — *스키마:* `{설비, 카테고리, 점검주차, 결과, 이상여부}` → 병합 풀어 카테고리 전파 → 가로(wide)→세로(long) → 색을 `이상여부` 값으로.
**③ 결과:**
```json
[
 {"설비":"CNC-01","카테고리":"윤활","점검주차":"1주차","결과":"정상","이상여부":false},
 {"설비":"CNC-01","카테고리":"윤활","점검주차":"3주차","결과":"이상","이상여부":true}
]
```
→ *무엇이 좋아졌나:* "한 행=한 점검"이 되어 이상 발생 설비·주차를 바로 집계하고, 색에만 있던 판정이 데이터로 남는다. (SOP 기준표 예시는 [5.2](../B-1%20데이터%20전처리.md#sec623).)

#### ㉤ Word·HWP 작업표준서(SOP)

**① 원본(가상)** — 제목이 스타일(제목1/제목2)로만 구분되고, 절차가 번호 목록, 중간에 주의 표가 끼임.
**② 전처리** — 문서 순서대로 순회하며 스타일→제목 계층(`#`/`##`)으로, 절차는 단계 리스트로, 표는 행/열로.

```json
{"chunk_id":"SOP-용접-03_2.1","heading":["용접 작업표준","2. 본작업","2.1 예열"],
```

#### ㉥ CSV·센서 로그 (설비 진동 시계열)

**① 원본(가상)** — cp949 인코딩, 머리글 위 장비정보 2줄, 단위 행, 값에 콤마·단위:
```
# 설비: PRESS-07   추출: 2026-06-15
timestamp,vibration,temp
[-],[mm/s],[℃]
2026-06-15 09:00:01,"1,024",47.5℃
2026-06-15 09:00:01,"1,024",47.5℃     ← 중복
```
**② 전처리** — 인코딩 감지→머리글 행 탐지→단위 행 제거→숫자 정제(콤마·℃ 제거)→타임스탬프 표준화·중복 제거→1분 평균 다운샘플.
**③ 결과:**
```json
[{"ts":"2026-06-15T09:00:00Z","설비":"PRESS-07","vibration":1024.0,"temp":47.5}]
```
→ *무엇이 좋아졌나:* 깨진 한글·중복·단위 섞인 값이 깔끔한 수치 시계열이 되어, "PRESS-07 진동 이상 추세"를 바로 분석한다.

**① 원본(가상)** — 발화자 표기 제각각, 연속 발화 줄바꿈, 시스템 줄:

```
[10:03] 김검사: 로트 0312 표면 결함이요
재현됩니다
[10:04] 이엔지(eng.lee): RT 다시 돌려보죠
```

**② 전처리** — 발화자·시각 분리, 별칭(eng.lee→이엔지) 정규화, 연속 발화 병합, 시스템 줄 제거.
**③ 결과:**
```json
[{"time":"10:03","speaker":"김검사","text":"로트 0312 표면 결함이요 재현됩니다"},
```

#### ㉧ 이미지·스캔본 (손글씨 섞인 검사 일지)

**① 원본(가상)** — 종이 검사 일지를 스캔한 이미지. 인쇄 항목명 + **손글씨 측정값**, 일부 흐림:
```
[스캔 이미지]  설비점검일지 (PRESS-07)
 점검일: 2026-06-15   점검자: (손글씨) 김O수
 항목        측정      판정
 유압        12.4MPa   양호       ← '12.4'는 손글씨, 흐릿
 윤활유량    (흐림)    ___        ← OCR 저신뢰
```
**② 전처리** — 보정(기울기·대비) → OCR(kor+eng) → **단어별 신뢰도**를 받아 낮은 것은 검수 플래그 → 표 구조 복원.
**③ 결과:**
```json
[
 {"항목":"유압","측정":"12.4MPa","판정":"양호","provenance":{"doc_id":"점검일지_PRESS-07_20260615"},"_review":false},
 {"항목":"윤활유량","측정":"[?]","판정":null,"_review":true}    ← 저신뢰 → 사람 검수로
]
```
→ *무엇이 좋아졌나:* 스캔 일지가 검색 가능한 레코드가 되되, **흐린/손글씨 셀은 자동 신뢰하지 않고 검수로 분리**돼 잘못된 값이 들어가지 않는다. (대량 스캔·디지털화 자체는 [F-3](../../F-3%20데이터%20디지털화/F-3%20데이터%20디지털화.md).)

### [Backup 3-F] 처음부터 끝까지 — 한 양식 완성 사례 (가상)

[5.2](../B-1%20데이터%20전처리.md#sec623)의 "스키마 먼저 → 값 추출"을 **현실적인 양식 하나로 끝까지** 펼친 튜토리얼이다. 실제 파일 대신 **가상의 양식**을 쓰되, 실무에서 부딪히는 지저분함(다중 행 머리글·병합·조건 섞임·색·소계 행)을 모두 넣었다. 그대로 따라 하면 된다.

#### 0단계 — 원본 양식 (가상: 두산에너빌리티 「용접 공정 검사 기준표.xlsx」)

```
시트 '용접검사기준'
(1행)       |       치수 기준        |      | 비파괴 |          ← 상위 그룹 머리글(병합)
(2행) 공정   | 검사항목 | 하한 | 상한 | 단위 | 검사주기 | 판정방법
──────────────────────────────────────────────────────────────
용접부 | 외관   |  -   |  -   |  -  | 매 로트 | 육안+PT
(병합) | 치수   | 99.9 |100.1 | mm  | 매 로트 | 캘리퍼
(병합) | 두께   |  8   |  12  | mm  | 일 1회  | UT (두께 10mm↑는 RT)     ← 조건이 문장에 섞임
도장   | 두께   |  80  | 120  | ㎛  | 일 1회  | 막후계  [노랑=주의]    ← 색에만 담긴 의미
──────────────────────────────────────────────────────────────
※ 합계 4건   비고: 동절기 예열기준 별도                            ← 데이터 아닌 소계/비고 행
```
함정 6가지가 한 표에: ① 머리글 2행(상위 그룹+컬럼) ② '용접부' 세로 병합 ③ `-`(해당없음) ④ 판정방법 안에 **조건**("10mm↑는 RT") ⑤ **색**("[노랑=주의]") ⑥ 맨 아래 **소계·비고 행**.

#### 1단계 — 사람의 단계 분해 (설계)

1. **한 건(레코드) = "한 (공정·검사항목)의 기준"** → 데이터 행은 4건(합계 행 제외).
2. 상위 그룹 머리글(치수 기준/비파괴)은 분류용일 뿐 → 실제 컬럼은 2행을 쓴다.
3. 병합된 **'용접부'는 치수·두께 행에 전파**.
4. `-`는 **null**(해당 없음).
5. 판정방법 안의 **조건은 `조건` 컬럼으로 분리**("두께 10mm↑는 RT").
6. **색 "[노랑=주의]"는 `주의여부`(true/false) 컬럼으로**.
7. **합계·비고 행은 버린다**.

#### 2단계 — Phase 1 프롬프트 (스키마 정의 요청)

```text
너는 데이터 전처리 설계자다. 아래 [표]를 보고, 한 건(레코드)을 이루는 '추출 항목 스키마'를
제안하라. 각 항목: 이름 · 의미 · 예시값 · 타입 · 필수여부.
규칙: (1) 표에 실제로 있는 정보만. (2) 병합된 상위 값(공정)은 하위 행에 적용된다고 적어라.
(3) 판정방법 문장에 조건이 섞여 있으면 별도 항목으로 분리하라. (4) 설명 없이 JSON으로만.
[표]
공정|검사항목|하한|상한|단위|검사주기|판정방법
용접부|외관|-|-|-|매 로트|육안+PT
용접부|치수|99.9|100.1|mm|매 로트|캘리퍼
용접부|두께|8|12|mm|일 1회|UT (두께 10mm↑는 RT)
도장|두께|80|120|㎛|일 1회|막후계 [노랑=주의]
```

**→ LLM이 제안한 스키마(가상 응답):**
```json
[
 {"name":"공정","의미":"검사가 속한 공정","예시":"용접부","타입":"string","필수":true},
 {"name":"검사항목","의미":"점검 항목","예시":"치수","타입":"string","필수":true},
 {"name":"기준_하한","의미":"허용 하한","예시":99.9,"타입":"number|null","필수":false},
 {"name":"기준_상한","의미":"허용 상한","예시":100.1,"타입":"number|null","필수":false},
 {"name":"단위","의미":"측정 단위","예시":"mm","타입":"string|null","필수":false},
 {"name":"검사주기","의미":"검사 빈도","예시":"매 로트","타입":"string","필수":true},
 {"name":"판정방법","의미":"검사 방법","예시":"캘리퍼","타입":"string","필수":true},
 {"name":"조건","의미":"적용 조건(있으면)","예시":"두께 10mm↑는 RT","타입":"string|null","필수":false}
]
```

#### 3단계 — 사람 검수 → 스키마 확정

LLM 제안에 색 컬럼이 빠졌다 → **`주의여부`(boolean) 항목 추가**. 단위·판정방법에 **표준값 목록**을 달아 확정한다(단위 ∈ {mm, ㎛, ℃, …} / 판정방법 ∈ {육안+PT, 캘리퍼, UT, RT, 막후계, …}). 이 **확정 스키마가 골든 기준**([2.4](../B-1%20데이터%20전처리.md#kq4))이 된다.

#### 4단계 — Phase 2 프롬프트 (값 추출)

```text
너는 데이터 추출기다. 아래 [스키마]에 따라 [표]의 모든 검사 기준을 JSON 배열로 추출하라.
규칙: (1) 스키마 항목만, 정확한 키로. (2) 원문에 없으면 null — 지어내지 마라.
(3) 병합된 공정명은 하위 행에 전파. (4) 판정방법 속 조건은 '조건'으로 분리, 본문엔 방법만 남겨라.
(5) '[노랑=주의]' 등 색 표시는 주의여부=true, 본문에서 제거. (6) '합계'·'비고' 행은 제외.
(7) 숫자·단위는 원문 그대로. (8) 설명 없이 JSON 배열만.
[스키마]
{ …3단계 확정 스키마… }
[표]
…(0단계 표)…
```

**→ LLM 추출 결과(가상 응답):**

#### 5단계 — 검증 (여기서 실제로 한 건 걸렸다)

- **행 수:** 표의 "합계 4건"과 대조 → 추출 4건 일치. *(첫 시도엔 LLM이 '합계' 행까지 넣어 5건이 나왔다 → 행 수 불일치로 걸려, Phase 2 규칙 (6)에 "합계·비고 행 제외"를 명시해 재실행.)*
- **필수값:** 공정·검사항목·검사주기·판정방법 빠짐 없음 ✓
- **표준값:** 단위·판정방법이 허용 목록 안 ✓ (벗어나면 사람 검수)
- **조건·색 분리 확인:** "UT (두께 10mm↑는 RT)"가 판정방법="UT" + 조건으로 분리됐고, 도장 행 주의여부=true ✓
