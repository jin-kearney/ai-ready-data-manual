
# 1. Why
현재
데이터가 아무리 많아도 AI가 읽지 못하는 형식이면 활용으로 이어지지 않는다. 계열사 인터뷰에서도 AI 과제의 초기 부담 상당수가 모델이 아니라 문서를 데이터로 바꾸는 단계에 몰려 있었다. 전처리는 이 부담을 표준화해, 쌓여 있지만 못 쓰던 문서를 AI가 바로 쓰는 데이터로 전환한다.


수정
데이터가 충분히 존재하더라도 AI가 활용할 수 있는 형태로 정리되어 있지 않으면 실제 활용에는 한계가 있다. 계열사 인터뷰에서도 AI 과제 초기 단계의 주요 업무는 모델 개발보다 데이터 수집, 정리, 구조화에 집중되어 있는 것으로 확인되었다. 데이터 전처리는 이러한 준비 과정을 체계화하여 AI 활용이 가능한 데이터 형태로 변환하는 역할을 수행한다.

### 1.2 전처리로 얻는 것

현재
- 문서 속 정보가 검색·분석 가능한 데이터로 살아난다.
수정
- 폴더에 잠자던 성적서 수천 건이 표를 행·열로 살리고 페이지 위치를 붙이는 순간 불량 원인 질의응답의 검색 자료가 된다.

현재
- 문서 기반 정보가 검색·분석 가능한 데이터로 전환된다.
수정
- 기존에 활용되지 못하던 성적서 데이터도 표 구조와 출처 정보가 정리되면 불량 원인 분석과 검색에 활용할 수 있다.

현재
- 근거를 추적할 수 있어 신뢰가 올라간다.
수정
- 근거를 추적할 수 있어 데이터 활용 신뢰성을 높일 수 있다.

현재
- 한 번 만든 변환 자산을 반복 재사용한다
수정
- 한 번 구축한 전처리 결과를 반복적으로 활용할 수 있다.

### 2.1 데이터 전처리란 + 범위·경계

현재
전처리는 이 숨은 구조를 복원해 넘긴다.

수정
전처리는 문서 내 구조 정보를 추출하여 AI가 활용 가능한 형태로 변환한다.

### 2.2 전처리 변환 사슬
현재

수정
구조화 과정이 수행되어야 AI가 특정 수치가 어느 항목에 해당하는지 해석할 수 있고

현재
단순 검색은 위치보존까지로 충분할 때도 있고, RAG는 청킹까지 간다.
수정

### 2.3 구조화 산출물

현재
JSON·Markdown·표는 그 구조를 담는 그릇일 뿐, 그릇 자체가 목적은 아니다.

수정
JSON, Markdown, 표는 구조화 결과를 표현하는 형식일 뿐이다.
전처리의 목적은 특정 형식을 만드는 것이 아니라 AI 활용에 필요한 구조와 출처 정보를 확보하는 것이다.

### 2.3 구조화 산출물

현재

수정
AI가 값의 의미와 위치를 해석할 수 있고

현재

수정
답변과 함께 "어느 보고서 몇 페이지"인지를 근거로 제시할 수 있다

수정

여기서 LLM은 답변 생성 모델이 아니라 문서를 구조화하는 자동화 엔진으로 활용된다.

### 5. Tech Stack — 전처리 솔루션 비교

현재
양식이 일정하고 대량인 문서에 쓴다.

수정
양식이 일정하고 대량인 문서에 주로 활용한다.

현재
한 솔루션으로 여러 포맷의 레이아웃·표를 인식한다. 최근 제품일수록 머신러닝·LLM을 안에 쓴다.

수정
하나의 솔루션으로 다양한 포맷의 레이아웃과 표 구조를 인식할 수 있다. 최근 솔루션은 머신러닝과 LLM 기술을 활용해 인식 정확도와 자동화 수준을 높이고 있다.

현재
병합·계층·제각각인 양식처럼 코드 파서·파싱 솔루션이 깨지는 곳을, 비전(이미지·표 인식)을 지원하는 범용 LLM에 "스키마 먼저 → 값 추출"로 시킨다.

수정
병합 구조나 계층 구조가 복잡하고 양식 편차가 큰 문서처럼 기존 파서나 파싱 솔루션만으로 처리하기 어려운 영역은, 비전(이미지·표 인식)을 지원하는 범용 LLM을 활용하여 "스키마 정의 → 값 추출" 방식으로 처리한다.

## 6. Where — 다른 주제와의 관계

현재
가장 헷갈리는 경계는 B-1과 F-3다. 둘 다 문서를 데이터로 바꾸지만, F-3는 종이·아날로그를 대량으로 디지털화(스캔·OCR)하는 일이고, B-1은 이미 디지털인 자료를 AI가 읽을 구조로 변환하는 일이다. F-3가 디지털로 만들기라면 B-1은 구조화하기다.

수정안
실무에서 가장 자주 혼동되는 영역은 B-1과 F-3의 경계이다. 두 주제 모두 문서를 데이터로 전환한다는 공통점이 있지만, F-3는 종이·아날로그 자료를 디지털 자산으로 전환하는 역할을 수행하고, B-1은 이미 디지털화된 자료를 AI가 활용할 수 있는 구조로 변환하는 역할을 수행한다. 즉, F-3가 디지털화 단계라면 B-1은 구조화 단계에 해당한다.
