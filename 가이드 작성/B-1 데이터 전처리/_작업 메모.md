## 정본(canonical) 모델 — 끝까지 일관 유지할 것

**전처리 변환 사슬 4단계: ① 추출(Extract) → ② 구조화(Structure) → ③ 위치보존(Provenance) → ④ 청킹(Chunk).**
- 3장(무엇을)에서 이 4단계를 정의하고, 5장 시나리오·6장 구축 파이프라인(6단계: 대상선정+위 4단계+적재)·7장 경계까지 이 모델로 일관 재사용했다.
- 6단계 구축 파이프라인 = 4단계 사슬 앞에 "①대상 선정", 뒤에 "⑥적재"를 더한 것. (3장 4단계와 6장 6단계가 어긋나 보이지 않게 — 6.1에서 "③~⑤는 3장에서 갖춘 것의 실행"이라고 명시함.)

## 주제 경계 (MECE) — 가장 중요

- **B-1 = 형식 변환만** (이미 디지털인 자료를 AI가 읽을 구조로).
- 대량 디지털화·OCR = **F-3**, 의미(정답) 라벨 = **B-2**, 사용 가능 판정 = **C-2**, 근거 추적 = **C-3**.
- ★ 가장 헷갈리는 경계: **B-1 vs F-3** — F-3는 "디지털로 만들기(스캔·OCR)", B-1은 "구조화하기". 7장에서 강조 박스로 명시.
- 데이터 준비 관점 고정: 청킹·벡터DB·임베딩을 다루되 "RAG/AI 만드는 법"이 아니라 "검색 가능한 청크로 데이터를 준비"하는 관점. 본문에 *(임베딩 모델 만드는 게 아님)* 류 단서를 명시해 둠.
