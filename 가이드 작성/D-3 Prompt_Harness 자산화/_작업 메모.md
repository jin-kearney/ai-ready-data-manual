# D-3 Prompt/Harness 자산화 — 작업 메모

이 주제 전용 결정·정본 모델·주의점·피드백 이력을 누적한다. (전역 규칙은 `공통 규칙/`·`CLAUDE.md`)

## 주제 정의

- D-3 정의: 사람의 업무 절차·판단 기준을 AI가 실행 가능한 Prompt·Harness로 구조화·관리하는 체계. (MECE 경계: 지시·판단 로직만)
- 선택형 주제 — Why와 When을 1장으로 통합(자주 쓰고 영향 큰 것만 자산화).

## 정본 모델 (문서 전체에서 일관 유지)

- **자산 목록(§2.2)** = 7요소: 시스템 지시문 · 업무 지시 Prompt · 역할 정의 · 판단 기준 · 출력 형식 · 금지 행동 · Tool 호출 순서. ("예시"로 명시 — 정해진 표준 개수 아님)
- **업무→Prompt 변환 틀(§2.3)** = 5구획: 역할 · 입력 · 판단 기준 · 금지 행동 · 출력 형식. (이 5구획을 SVG·Before/After·별첨 템플릿까지 일관 사용)
- **Harness(§2.4)** = 4요소: Prompt 자산 + 입력 데이터 연결 + Tool 호출 순서 + 출력 형식·검증.
- **버전 상태(§5.2)** = 4단계: 후보 → 검토중 → 배포 → 폐기 (롤백은 직전 배포 버전으로). 시맨틱 버전 X.Y.Z.
- **구축 절차(§5.1)** = 6단계: 대상 선정 → Prompt 변환 → 의존성 연결 → 테스트·평가 → Harness 패키징 → Registry 등록·배포.
- 용어 통일: **Registry(등록소)** 최초 1회 병기 후 Registry. Prompt·Harness(하네스)는 영문 유지. 시맨틱 버전(Semantic Versioning)·회귀 테스트(Regression Test) 최초 1회 병기.
- End-to-End 사례: **두산전자 CCL 클레임 원인 분류**(소재/공정 구분) 하나로 §5 전체를 관통(작업지시 0622 규칙).
