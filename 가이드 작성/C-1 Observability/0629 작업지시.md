# B-1,2,3 양식을 참조
> 전반적으로 이 가이드 특성상 필요한 내용이 아닐 경우 B-1,2,3에 없는 것을 포함하지 않는다. 문장 구조나 어투, 배치 위치 최대한 B-1,2,3 스타일에 맞춰서.

# 목차 수정

Why --> What --> When (있을 시) --> How --> Tech Stack --> 다른 주제와의 관계 (Where) 로 유지

없는 섹션은 포함 안해도 무방, 하지만 Why What How Tech Stack은 필수로 포함하며 순서도 동일하게 유지함


제목에 "(한눈에)" 이런 문구 삭제

윗단의 "예시 표기 안내" --> 삭제

현재:

## 4. 예시 시나리오 (한눈에)
### 4.1 적용 전과 후
### 4.2 흐름 미리보기

수정:

## 4. How — 어떻게 준비·운영하나
### 4.1 적용 전후 비교
### 4.2 구축 절차

또는 예시를 별도 장으로 두지 않고 How 안으로 흡수한다.

How를 다음과 같이 변경 (예시 시나리오 -> How로 흡수) 
## 4. How — 어떻게 준비·운영하나 ### 4.1 적용 전후 비교 ### 4.2 Observability 구축 절차 ### 4.3 관측 지표·판단 기준 설정 ### 4.4 알림·에스컬레이션 운영 ### 4.5 운영 성과 측정


현재:

## 7. Where — 다른 주제와의 관계

수정:

## 6. 다른 주제와의 관계

“Where” 표기는 목차 기준에서 제외하고, 실제 제목은 B-1/B-2/B-3와 맞춰 한글 중심으로 정리한다.

8. KPI - 성과지표를 --> 4.7 운영 성과 측정으로 빼고

KPI 항목은:
- 이상 감지 시간 (MTTD)
- 해결시간 (MTTR)
- 놓친 건수
- 오탐률
- 관측 커버리지

로 유지하되, 산식이 산정 가능한 경우 표에 칼럼을 추가


문체수정: 보고서 톤 사용

막히는 지점, 값이 튀어도, 조용히 망가진다, 엉뚱한 곳을 뒤지는 시간, 누가 볼 것인지
--> 이런 워딩 금지, 대신 현업 적용 시 발생하는 문제, 이상값이 유입됨, 운영 중 비정상 상태가 발생함, 원인 분석 시간이 증가함, 담당 책임이 불명확함 등의 보고서 톤 사용
