# md → Word 가이드 변환 스킬

`.md` 가이드를 가독성 높은 `.docx`로 변환한다. 핵심은 세 가지다 — **Mermaid 다이어그램
이미지화**, **표 테두리·가독성**, **링크 보존**.

## 무엇을 해주나 (Kearney 참조 워드 서식에 맞춤)

- **글씨 크기**: 본문 `BODY_PT`(10pt). 제목은 참조본과 같은 크기로 고정 — **H2 12pt,
  H3·H4 10pt**(H4는 굵게·비기울임). 본문 위계에서 단계별로 작아진다.
- **폰트**: 본문·제목·표·코드까지 **Noto Sans KR**로 통일(`FONT`). 코드/JSON 예시도
  같은 폰트로 맞춘다(참조본과 동일).
- **본문 정렬**: 본문 문단을 **양쪽 정렬(justify)** 한다(`JUSTIFY`). 표 셀·제목은 제외.
- **다이어그램 캡션 없음**: 이미지에 "다이어그램 N" 같은 캡션을 달지 않는다(빈 alt +
  `-implicit_figures`).
- **다이어그램**: 본문의 ```mermaid``` 블록을 헤드리스 Chromium + mermaid-cli로 PNG 렌더링
  (원본 파랑/초록 컬러 스키마·한글 라벨 그대로). 페이지를 넘는 세로형은 높이 8.4in, 가로형은
  너비 6.25in 안으로 자동 축소. **(이 렌더링/이미지 처리 부분은 예전 스킬 그대로 — 미수정)**
- **표**: pandoc이 준 비례 열 너비를 그대로 쓰고(없으면 직접 계산), **모든 칸에 테두리**,
  **머리글 행 음영(연파랑 DCE8F5)+굵게**, 셀 세로 가운데 정렬, **행 높이 압축**(셀 문단 여백 0),
  표를 본문 폭의 96%(`TABLE_PCT`)로 약간 인셋해 **가운데 정렬**(`CENTER_TABLES`)한다.
- **링크**: 외부 URL(W3C·arXiv 등), 인접 가이드/별첨 `.md` 링크, 문서 내 목차·섹션 점프 링크
  (수기 `<a id="...">` 앵커 포함)를 모두 Word 하이퍼링크/책갈피로 변환.
- **인용 블록(`> …`)**: 들여쓰기 대신 **테두리 박스**(연파랑 테두리 + 옅은 음영)로 렌더링한다
  (`boxify_blockquotes`). 연속된 인용 줄은 Word가 한 박스로 합친다. 색은 `BOX_BORDER`/`BOX_FILL`.

## 실행 절차 (Claude가 따른다)

1. **환경 준비**: 먼저 렌더링 환경을 부트스트랩한다.
   ```bash
   bash "<이 스킬>/scripts/setup_env.sh"
   ```
   - Chromium을 분할 다운로드(샌드박스 프록시가 대용량 단일 다운로드를 끊음)하므로,
     `SETUP INCOMPLETE`가 나오면 **같은 명령을 다시 실행**한다(이어받기 됨). `SETUP OK`가
     나올 때까지 반복. 한 번 깔리면 이후엔 즉시 통과.
2. **변환**: 소스 .md와 출력 .docx 경로로 엔진을 실행한다.
   ```bash
   python3 "<이 스킬>/scripts/md2docx.py" "<원본>.md" "<출력>.docx"
   ```
   - 항상 **현재 원본 파일을 다시 읽는다**(작업 중 원본이 갱신될 수 있음).
   - 출력은 프로젝트의 `word 버전 매뉴얼/` 폴더에 같은 이름으로 저장한다.
3. **검증**: LibreOffice로 PDF 변환 후 1~2장을 이미지로 떠서 표 테두리·다이어그램·한글이
   제대로 나오는지 눈으로 확인한다. (미리보기에서 `§`가 `$`로 보이는 것은 LibreOffice
   폰트 대체일 뿐 — 실제 docx에는 `§`가 정상 포함됨.)
4. **전달**: `present_files`로 결과 .docx를 공유한다.

## 사용자가 조정할 수 있는 스타일 (md2docx.py 상단 상수)

요청 시 아래 값을 바꿔 재변환한다.

| 항목 | 상수 | 기본값 | 의미 |
|------|------|--------|------|
| 통일 폰트 | `FONT` | Noto Sans KR | 본문·제목·표·코드 폰트 |
| 본문 글씨 크기 | `BODY_PT` | 10 | 기본 본문 pt |
| 제목 글씨 크기 | `H2_PT`/`H3_PT`/`H4_PT` | 12/10/10 | 섹션/소/소소 제목 pt(H4는 굵게) |
| 본문 양쪽 정렬 | `JUSTIFY` | True | 본문 문단 justify(표·제목 제외) |
| 표 너비 | `TABLE_PCT` | 4800 | 본문 폭의 백분율×50 (4800=96%) |
| 표 가운데 정렬 | `CENTER_TABLES` | True | 표를 페이지 가운데로 |
| (빈 그리드용) 표 폭/최소 열 | `TOTAL`/`MINW` | 9180/1100 | pandoc이 열 너비를 안 줄 때만 사용 |
| 테두리 색/두께 | `BORDER_COLOR`/`BORDER_SZ` | 808080/4 | 표 선 색(hex)/두께(1/8pt) |
| 머리글 음영 | `HEADER_FILL` | DCE8F5 | 머리글 배경(hex) |
| 박스 테두리/배경 | `BOX_BORDER`/`BOX_FILL` | 6B9AD1/EEF4FB | 인용 박스 색(hex) |
| 그림 최대 높이/너비 | `MAXH`/`MAXW` | 8.4in/6.25in | EMU(914400=1in) — 예전 그대로 |

## 워드에서 제외하는 섹션 (보일러플레이트)

아래는 Word 출력에서 **제거**한다(목차 링크도 함께 제거). `drop_sections`에서 처리.

- **"이 가이드가 답하는 N가지 질문"** 인용 박스 (`QUESTION_MARKERS`로 식별)
- **변경 이력 / 피드백 반영** 섹션

**참고자료(References)는 유지**한다. 본문의 `[N]` 표시는 클릭하면 해당 참고 항목으로 점프한다
(목록의 인라인 앵커 `<a id="refN">`를 pandoc 책갈피로 변환 → `convert_inline_anchors`).
다시 제거하고 싶으면 `_drop_heading`에 참고/References 패턴을 추가하면 된다.

## 마크다운 잔재 정리 (실제 워드 문서처럼)

- **굵게(`**...**`)**: pandoc `markdown` 리더를 쓴다(gfm 아님). gfm/CommonMark는 굵게가
  한글 조사 앞(`**점(노드)**으로`)에서 닫히지 못해 `**`가 그대로 남는데, `markdown` 리더는
  정상 처리한다.
- **YAML 머리말**(`--- title: ... ---`)은 본문에 안 찍히고 깔끔한 제목으로 처리된다.
- **콜아웃 표식**(`> [!question]`, `> [!note]` 등)은 변환 전 제거한다(`clean_markdown`).
- **목록 앞 빈 줄**: pandoc `markdown`은 목록 앞에 빈 줄이 없으면 목록으로 인식하지 못해
  `**제목**` 문단에 빨려 들어간다. 목록 앞에 빈 줄을 자동 삽입한다(`ensure_list_blanklines`).
- **표 셀 안 `<br>` 줄바꿈**: pandoc은 raw `<br>`을 버려 불릿이 한 줄로 붙는다. 변환 전
  `<br>`을 토큰으로 바꾸고, 변환 후 XML에서 실제 줄바꿈(`<w:br/>`)으로 되살린다. (토큰은
  `@` 등 마크다운 특수문자를 피한다 — `@`는 pandoc citations로 오인됨.)
- **목차/섹션 점프 링크**: `markdown` 리더는 제목 id에서 앞 번호를 떼므로(`1-제목`→`제목`),
  GitHub식 id를 따로 계산해 책갈피로 주입한다. Word 책갈피는 숫자로 시작 못 하므로 pandoc과
  동일하게 `X+sha1` 규칙으로 이름을 맞춘다 → `#1-…` 링크도 정상 점프.

## 메모

- 변환 엔진은 외부 의존 없이 `pandoc`(markdown 리더) + XML 후처리로 동작한다. python-docx를 쓰지 않는다.
- 검증기(docx validate.py)가 상대경로 `.md` 하이퍼링크를 "broken reference"로,
  Override로 선언된 png를 "미선언"으로 표시할 수 있으나 **둘 다 오탐**이며 Word에서 정상 열린다.
