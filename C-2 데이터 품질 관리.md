## C-2 데이터 품질 관리  *(기본적으로 다 필요한 주제 — "어디부터 게이트를 거나"가 핵심)*
> 한 줄: 데이터를 AI에 넣기 전 "이거 써도 되는가"를 판정하고, 쓸 자격(접근 권한)까지 통제하는 마지막 관문.
> 핵심 계열사: 두산퓨얼셀

```
1. 개요
   1.1 데이터 품질 관리란 (AI 투입 전 "쓸 수 있는가"를 판정하는 통제 관문)
   1.2 적용 범위와 체계 내 위치 (사용 판정·접근 차단이 C-2, 마스킹은 보안(F-4), 이상 감지는 Observability(C-1)와 구분)
2. 왜 필요한가
   2.1 현업 Pain Point (빈칸·오류·오래된 데이터가 그대로 들어가 AI 답이 틀리고, 봐선 안 될 데이터까지 학습됨)
   2.2 기대 효과 (믿을 수 있는 데이터만 AI에 들어가 답 신뢰도↑·사고 위험↓)
3. 무엇을 갖추나
   3.1 품질 기준 (빠짐없음·정확·일관·최신·유효 — 업무별 합격선)
   3.2 사용 금지 기준 (출처 불명·미승인·기준 미달·목적 외 데이터는 AI 사용 제한 — KQ2)
   3.3 접근 권한 정책 (역할·목적·민감도에 따라 조회/분석/AI 학습/반출 가능 여부 구분 — KQ3)
   3.4 예외 승인 항목 (기준 미달인데 꼭 써야 할 때: 승인자·사유·기간·범위 기록 — KQ5)
4. 게이트 대상 우선순위   ← 다 필요하므로 "어디부터 게이트를 거나"
   4.1 기본 필수 (AI 학습·RAG·자동화로 바로 투입되는 데이터부터 게이트)
   4.2 적용 판단 (틀리면 영향이 큰 데이터일수록 기준을 엄격히)
5. 예시 시나리오 (한눈에)   ← How 앞
   5.1 적용 전/후 (오래된 검사 데이터가 섞여 AI가 엉뚱한 판정 → 게이트를 걸면 기준 미달 데이터는 투입 전 자동 차단)
   5.2 흐름 미리보기 (품질 기준·권한 정해두기 → 투입 직전 자동 점검(게이트) → 통과만 AI로, 미달은 차단·예외 승인)
6. 어떻게 준비·운영하나 (상세)
   6.1 품질 점검 도구 검토 (자동 검사·게이트 연동)
   6.2 구축 절차 ★ 룰 기반 자동 판정 우선 (사람이 일일이 보지 않게, 규칙으로 자동 합/불 판정 → 애매한 것만 사람)
   6.3 운영 (기준·권한 갱신, 예외 승인 관리) / 역할·책임
7. 다른 주제와의 관계
   7.1 접근 차단(C-2)·마스킹(F-4 보안)·이상 감지(C-1)와의 경계
8. 성과 지표·고도화
   8.1 성과 지표 (품질 기준 통과율·권한 위반 건수·미승인 사용 차단 건수·개선 리드타임)
   8.2 고도화 (자동 판정 규칙 비중↑ → 사람은 예외만 검토)
```
