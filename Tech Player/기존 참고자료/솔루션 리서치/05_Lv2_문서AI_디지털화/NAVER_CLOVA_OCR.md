## 주요 기능

### 범용 OCR (General OCR)
- 인식 언어: **한국어, 영어, 일본어** (기본 지원)
- **수기체(필기) 인식**: 한국어·일본어 필기체 전용 모델
- 한자 지원: 자주 사용되는 한자 포함
- 이미지 품질 보정(기울기 보정, 노이즈 제거)
- 단어·줄·문서 단위 좌표 및 신뢰도 반환

### Document OCR (특화 모델)
현재 제공 중인 특화 모델:

| 모델 | 설명 |
|---|---|
| 영수증(Receipt) | 매출·매입 영수증 필드 자동 추출 |
| 신용카드 | 카드번호·유효기간·명의인 추출 |
| 사업자등록증 | 사업자 정보 구조화 추출 |
| 명함 | 이름·직함·연락처·이메일 추출 |
| 신분증(ID) | 주민등록증·운전면허증 정보 추출 |

> 특화 모델 종류는 지속 확대 예정

### Custom OCR (커스텀 모델 학습)
- 사용자 정의 도메인 데이터로 커스텀 OCR 모델 학습 가능
- 제조업 검사표·작업지시서 등 사내 특수 서식 대응

### API 형태
- REST API (이미지 URL 또는 Base64 인코딩 입력)
- 동기·비동기 처리
- JSON 출력 (텍스트, 좌표, 신뢰도)

## 강점

- **한국어 정확도 1위**: 국내 OCR 솔루션 중 최고 수준(97~99%), 2026년 평가 기준
- **수기체 한국어 전문**: 손글씨 검사표·서명·작업 기록 인식에서 경쟁사 대비 우수
- **한자 혼용 문서**: 한국 기업 문서에 빈번한 한자 지원
- **공공기관 대응**: Gov-NCLOUD 채널로 공공 클라우드 컴플라이언스 충족
- **국내 규정 준수**: ISMS-P 인증, 개인정보보호법 대응
- **HWP 문서 흐름**: NAVER 클라우드 내 다른 AI 서비스(Document)와 연계

## 약점·주의점

- **다국어 제한**: 영어·한국어·일본어 이외 언어 지원 미흡 (글로벌 문서에 부적합)
- **레이아웃 분석 제한**: 복잡한 다단 레이아웃·표 구조 분석에서 Azure/Upstage 대비 제한
- **표 추출(TSR)**: 전용 TSR 기능 제한 — 복잡 표 구조 처리는 별도 후처리 필요
- **RAG 청킹 미내장**: 문서 청킹 기능 별도 구현 필요
- **LangChain/LlamaIndex 공식 통합 부재**: 직접 REST API 통합 필요
- **온프레미스**: 일반 상용은 클라우드 전용, 온프레미스는 별도 협의

## 연동 생태계

```
NAVER Cloud: Object Storage, API Gateway, Cloud Functions
REST API: 이미지 URL/Base64 입력, JSON 출력
Python: requests 라이브러리로 직접 호출 (공식 예제 제공)
Gov-NCLOUD: 공공기관 전용 채널
커스텀 모델: NAVER AI Studio 내 학습 워크플로
```

## 최신 동향 (2025~2026)

- **2026 기준**: 국내 한국어 OCR 정확도 1위 평가 유지 (97~99%, lido.app 비교 기준)
- **지속**: 수기체 한국어 인식 모델 개선 — 손글씨 검사표·서명 대응
- **지속**: Document OCR 특화 모델 종류 확대 예정 (사업자등록증, 의료 문서 등)
- **커스텀 모델**: 사용자 정의 도메인 학습 기능 강화
- **손글씨 OCR 평가**: 한글 필기 인식에서도 국내 최고 수준 (lido.app 2026년 비교 기준)

## 제조업 관점 코멘트

- **수기 검사표**: 한국어 수기체 인식 97~99%는 현장 작업자 손글씨 검사 결과 디지털화에 직접 활용 가능
- **작업지시서(WI)**: 혼합 인쇄·수기 양식의 작업지시서 OCR에 최적
- **품질 성적서**: 한자 혼용 품질 문서(구형 양식) 처리 시 한자 지원 강점
- **커스텀 모델**: 회사 특유의 내부 서식(검사 코드·약어·특수기호)에 대한 도메인 학습 가능
