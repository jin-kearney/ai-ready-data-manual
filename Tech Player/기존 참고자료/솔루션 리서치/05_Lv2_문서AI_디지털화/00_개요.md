# 구분 A: Lv.2 문서 AI·디지털화 — 개요

## 1. 구분 정의

**문서 AI·디지털화**는 PDF·PPT·Excel·도면·수기문서 등 비정형 문서를 AI가 읽고 활용할 수 있는 **구조화 데이터로 변환**하는 기술 영역이다.

| 하위 주제 | 코드 | 설명 |
|---|---|---|
| 비정형 문서 파싱/전처리 | B-1 | 디지털 문서(PDF·DOCX·PPT·Excel)의 레이아웃 분석, 표 추출, 청킹, RAG 파이프라인 전처리 |
| 종이·수기 OCR 디지털화 | F-3 | 스캔 이미지·수기 문서·도면·폼에서 텍스트·수치·구조를 추출, 디지털 레코드化 |

두 주제는 실무에서 중첩되며, 대부분의 솔루션이 두 영역을 함께 지원한다.

### 5-2. 제조업 현장 관점 코멘트

제조업 현장에서 특히 중요한 문서 유형은 **수기 검사표, 품질 성적서, P&ID 도면, 설비 매뉴얼, 작업지시서(WI)** 등이다.

- **수기 검사표·체크시트**: NAVER CLOVA OCR(한국어 수기체 97~99%)과 Azure Document Intelligence(수기 인식 최강)의 조합이 유리. AWS Textract도 회전 텍스트·저해상도 팩스 개선(2026).
- **P&ID·기계 도면**: 전용 도면 AI 솔루션(예: LandingAI ADE)이 별도로 존재. 범용 Document AI 중 Google Document AI(Gemini 3 Flash/Pro 기반 레이아웃 파서)가 그림·기호 인식에 유리.
- **품질 문서(SPC 표, 측정 성적서)**: 표 추출(TSR)이 핵심. Unstructured.io(Table Score 0.844), Upstage Document Parse(HTML 변환), Azure Document Intelligence 모두 우수.
- **온프레미스 강제 환경(보안 구역, 생산망 분리)**: ABBYY Vantage와 Unstructured.io OSS 버전이 최적. Azure/AWS는 컨테이너 배포 가능하나 인터넷 연결 필요.
- **ERP/MES 통합(SAP, 두산 DART)**: Azure(M365/Teams 연계), AWS(S3·Lambda 연계)가 기존 IT 인프라와 결합 용이. ABBYY Vantage는 SAP 커넥터 보유.

### 5-3. 비용 구조 포인트

- 소량(~1만 페이지/월): Upstage Document Parse, Unstructured.io 서버리스 API 추천 (소량 종량제)
- 중량(10~100만 페이지/월): AWS Textract(볼륨 할인), Google Document AI 고려
- 대량 기업 내재화: ABBYY Vantage 연간 라이선스, Unstructured.io 엔터프라이즈 무제한 플랜
