# Lv.2 OT/IoT 수집 — 솔루션 개요

> **커버 AI-Ready Data 주제**: D-1 Physical 데이터 (Edge Gateway, 시계열 표준화, OT/IT 망연계)
> **목적**: 공장 설비·센서·PLC 데이터를 AI 입력 품질로 수집·컨텍스트화·전달

---

## 1. 구분 정의

OT/IoT 수집 계층은 공장 바닥(Shop Floor)의 원천 데이터를 AI/분석 계층이 소비할 수 있는 형태로 만드는 **데이터 파이프라인 최전선**이다. 핵심 역할은 다음 세 가지다.

1. **프로토콜 수렴**: OPC UA, Modbus, MQTT, EtherNet/IP, PROFINET 등 이종 OT 프로토콜을 표준 메시지 포맷으로 변환
2. **Edge 처리**: 망분리 환경에서 로컬 필터링·집계·이상 탐지를 수행하여 클라우드 대역폭·비용 절감
3. **시계열 컨텍스트화**: 태그(Tag) 기반 원시 수치에 설비 계층(Asset Hierarchy), 단위(UoM), 타임스탬프 정규화를 부여해 AI가 바로 쓸 수 있는 데이터로 변환

### 간략 언급 솔루션
- **TimescaleDB**: PostgreSQL 확장 시계열 DB. IoT 데이터를 비즈니스 데이터(ERP·MES)와 SQL JOIN이 필요한 시나리오에 유리. Telegraf와 조합 가능.
- **Litmus Edge**: 250+ 산업 프로토콜 지원 Edge 플랫폼. AWS IoT SiteWise와 공식 통합(2023~). OT 프로토콜 폭이 가장 넓어 레거시 PLC 연결에 특화.

## 4. 선택 가이드

### 4.1 시나리오별 추천

| 시나리오 | 추천 솔루션 | 이유 |
|---|---|---|
| 레거시 PLC 다수 + 망분리 On-prem | **Ignition** 또는 **AVEVA PI System** | 다양한 드라이버, 내장 Historian, 오랜 제조업 레퍼런스 |
| 클라우드 우선, Azure 기반 | **Azure IoT Operations + Fabric RTI** | OT 엣지→클라우드 일관 파이프라인, Copilot/AI 통합 |
| AWS 기반, 매니지드 서비스 선호 | **AWS IoT SiteWise** | 태그·에셋 모델 내장, 250+ 프로토콜(Litmus 연동) |
| 시계열 데이터 실시간 스트리밍 우선 | **Confluent (Kafka) + HighByte** | Kafka 스트림 + HighByte 컨텍스트화 조합, Unified Namespace |
| 오픈소스 최소 비용, 개발팀 주도 | **InfluxDB + Telegraf** | MIT 라이선스, 400+ 플러그인, SQL-like Flux 쿼리 |
| 다계열사 OT 데이터 표준화 통합 | **HighByte Intelligence Hub** | 코드리스 데이터 모델링·거버넌스, 멀티 사이트 확장 |

### 4.2 제조업 대기업(다계열사 공장, OT 보안 망분리) 관점

> **망분리 환경 핵심 고려사항**
>
> 한국 대기업 계열사 공장은 통상 **OT 네트워크(레벨 0~2)와 IT 네트워크(레벨 3~4)가 물리적 또는 논리적으로 분리**되어 있다. 이 환경에서 데이터를 AI 플랫폼으로 올리려면 단방향 데이터 다이오드, DMZ 게이트웨이, 또는 MQTT 브로커 중계 방식을 채택한다.

| 고려 요소 | 코멘트 |
|---|---|
| **단방향 게이트웨이** | AVEVA PI System의 PI Interface, Ignition의 Edge 모듈, HighByte가 모두 단방향 전송(읽기 전용 에이전트) 지원. 클라우드 솔루션(Azure, AWS)은 MQTT outbound-only 구성으로 보완 |
| **다계열사 표준화** | HighByte 또는 AVEVA AF(Asset Framework)의 정규화 모델이 다계열사 간 태그명·단위 통일에 적합. Ignition의 UDT(User Defined Type)도 유사 역할 |
| **AI Ready 데이터 품질** | 원시 태그 데이터를 AI가 쓸 수 있으려면 타임스탬프 동기화·결측값 처리·컨텍스트(설비 계층) 부여가 필수. HighByte, AVEVA PI+AF, AWS SiteWise Asset Model이 이 역할에 특화 |
| **온프레미스 규제 데이터** | 산업 기밀·설계 도면이 포함된 경우 클라우드 전송 제한. Ignition On-prem + InfluxDB 로컬 조합이 안전 |
| **비용 통제** | 클라우드 솔루션의 태그 수(포인트)·메시지 수 기반 과금은 수백만 태그 환경에서 급증. AVEVA PI System 구독 모델 총소유비용(TCO)과 클라우드 종량을 비교 분석 권장 |
