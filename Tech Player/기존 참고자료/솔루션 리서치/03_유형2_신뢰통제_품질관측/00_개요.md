# 유형② 신뢰 통제 — 데이터 품질·Observability 전문 솔루션 개요

> 작성일: 2026-06-10 | 커버 AI-Ready Data 주제: C-1 Observability, C-2 품질관리, C-3 Lineage

---

## 1. 구분 정의

유형② 신뢰 통제 솔루션은 데이터 플랫폼(Snowflake·Databricks·BigQuery 등)에 내장된 기본 품질 기능을 **보강**하여,
프로덕션 데이터의 이상·결함을 자동으로 감지·추적·알림하고 근본 원인(Lineage 포함)까지 연결하는 **독립형 전문 계층**이다.

| AI-Ready Data 주제 | 역할 |
|---|---|
| **C-1 Observability** | 신선도(Freshness)·볼륨·분포·스키마 변화 모니터링 |
| **C-2 품질관리** | 룰 기반/ML 자동 품질 체크, 데이터 계약(Contract) 집행 |
| **C-3 Lineage** | 컬럼 수준 계보 추적 → 영향 범위 분석·RCA |

## 5. 선택 가이드

### 5-1. 사용 시나리오별 추천

| 시나리오 | 추천 솔루션 | 이유 |
|---|---|---|
| Snowflake/Databricks 중심, ML 자동 감지 최우선 | Monte Carlo | ML-first 이상탐지, 50+ 커넥터, Lineage 통합 |
| 파이프라인 코드화·데이터 계약 중심 | Soda | SodaCL YAML, dbt/Airflow 깊은 통합, OSS 코어 |
| 개발자 주도 테스트 자동화, 비용 절감 | GX Cloud | ExpectAI 자동 생성, Data Asset 기반 예측 가능 비용 |
| 비정형 데이터(PDF·계약서) + 구조화 데이터 이중 커버 | Anomalo | LLM 기반 비정형 품질, AI 에이전트 RCA |
| 영향 범위 분석 + AI 신뢰 플랫폼 강조 | Bigeye | Lineage 기반 RCA, PII 자동 탐지, AI Trust 포지셔닝 |
| Hadoop/온프레미스 + 클라우드 하이브리드 환경 | Acceldata | Pulse(Hadoop), Torch(품질), Agentic ADM 플랫폼 |

### 5-2. 제조 대기업(다계열사) 관점 코멘트

**다계열사 제조 그룹** (예: 중공업·건설기계·방산·전자 등 이종 산업 계열사 공존) 환경에서는 다음 관점이 중요하다:

1. **하이브리드 데이터 환경**: 계열사별로 레거시 온프레미스 ERP/MES와 클라우드 DW가 혼재한다. Acceldata(Hadoop·온프레미스 지원)와 Soda(OSS CLI로 어디서나 실행)가 유연성에서 우위를 가진다.

2. **공정 데이터·센서 데이터 품질**: 실시간 스트리밍 이상탐지가 필요한 제조 현장에는 Monte Carlo(ML 자동탐지, Kafka 소스 지원)나 Bigeye(70+ 메트릭 라이브러리)가 적합하다.

3. **계열사 간 데이터 계약(Data Contract)**: 계열사 경계에서 데이터 SLA를 명문화하고 파이프라인 게이트로 집행하려면 Soda(SodaCL 계약 엔진)가 가장 성숙하다.

4. **AI 프로젝트 데이터 신뢰**: LLM 파이프라인용 비정형 문서(정비 보고서·시방서)의 품질까지 커버해야 한다면 Anomalo가 현재 유일한 선택지다.

5. **비용 예측성**: 계열사 수가 많으면 테이블·소스 수 기반 견적이 급등할 수 있다. GX Cloud(Data Asset 기반)와 Soda(SPU 단위)가 비용 예측이 용이하다.

6. **그룹 통합 관제**: 단일 대시보드로 다계열사 데이터 건강도를 관리하려면 Monte Carlo 또는 Acceldata의 멀티 소스 중앙화 기능이 요구된다.
