# Lv.2 AI 데이터 보안 — 솔루션 개요

> **커버 AI-Ready Data 주제**: F-4 AI 데이터 보안 (민감정보 탐지·마스킹·가명화, 비정형·Prompt·Agent 로그 커버, 플랫폼 내장 마스킹의 보강)
> **목적**: AI 파이프라인 전 구간(데이터 수집→학습→추론→에이전트 로그)에서 개인정보·기업 기밀 유출을 방지하고, 한국 개인정보보호법(PIPA) 및 글로벌 규제를 준수하면서 AI 활용 가능성을 보전한다.

---

## 1. 구분 정의

AI 데이터 보안 계층은 세 가지 방어선으로 구성된다.

1. **탐지(Discovery)**: 정형 DB·비정형 문서·AI 학습 데이터셋·LLM Prompt/응답에서 개인정보(PII)·기밀·민감정보 자동 식별
2. **보호(Protection)**: 마스킹·토큰화·가명화·암호화·합성 데이터 생성으로 원본 민감정보를 대체. AI 분석 유용성 보전
3. **접근 통제(Governance)**: 정책 기반 데이터 접근 제어(ABAC), Prompt 방화벽, Agent 로그 보안으로 AI 워크플로 전반의 데이터 거버넌스 시행

## 4. 한국 개인정보보호법(PIPA) 관련 맥락

| PIPA 요건 | 관련 기능 | 적합 솔루션 |
|---|---|---|
| 개인정보 수집·이용 목적 제한 | 데이터 분류·목적 태깅, 접근 정책 | BigID, Securiti.ai, Immuta |
| 가명정보 처리 기준 (2024 개정 가이드라인) | 비정형 데이터 가명화(이미지·음성·텍스트), k-익명성 | Presidio(비정형 특화), Protegrity(정형 특화) |
| 생성형 AI 개발·활용 안내서 (2025.08 PIPC) | Prompt·AI 출력 PII 탐지·차단 | Presidio + Securiti LLM Firewall, Privacera PAIG |
| 유출 사고 72시간 통지 | DSPM(데이터 보안 포지션 관리) 알림 | BigID, Securiti.ai |
| 위반 시 매출액 3% 과징금 | 컴플라이언스 리포팅·증빙 | Securiti.ai (100+ 규제 매핑), BigID |

> 한국 PIPA는 EU GDPR 수준의 규제 강도를 가지며, 2025년 생성형 AI 안내서 발표 이후 AI 시스템에서의 개인정보 처리에 대한 구체적 요건이 제시됨. AI 파이프라인 도입 시 PIPA 준수 체계를 사전에 설계해야 한다.

## 5. 선택 가이드

### 5.1 시나리오별 추천

| 시나리오 | 추천 솔루션 | 이유 |
|---|---|---|
| LLM/GenAI 서비스 구축, Prompt 보호 우선 | **Securiti.ai** 또는 **Presidio+커스텀** | LLM Firewall, 오픈소스 유연성 |
| 데이터 플랫폼(Snowflake/Databricks) 접근 통제 | **Immuta** 또는 **Privacera** | 플랫폼 내장 정책, 동적 마스킹 |
| 전사 민감 데이터 전수 탐지·분류 | **BigID** | 100+ 데이터 소스 커넥터, AI 기반 분류 |
| 정형 DB 토큰화·암호화 최고 성능 | **Protegrity** | 필드 수준 vaultless 토큰화, 고성능 |
| 오픈소스 기반 비용 절감, 개발팀 주도 | **MS Presidio** | MIT 라이선스, Python 통합, LLM 파이프라인 |
| 다국가 규제(GDPR+PIPA+CCPA) 통합 관리 | **Securiti.ai** | 100+ 규제 프레임워크 내장 |

### 5.2 제조업 대기업(다계열사) 관점 코멘트

> **제조 AI 파이프라인의 데이터 보안 특수성**
>
> 대기업 계열사 AI 프로젝트에서는 공장 센서 데이터(PII 無)보다 **임직원 데이터, 고객 정보, 설계 도면, 공정 노하우** 등이 AI 학습 데이터로 유입될 때 보안 위험이 발생한다.

| 고려 요소 | 코멘트 |
|---|---|
| **Copilot/GenAI 사용자 도구 도입** | 임직원이 사내 문서를 LLM에 붙여넣는 행위 → Presidio(프록시) 또는 Securiti LLM Firewall로 Prompt 스캔·차단 |
| **AI 학습 데이터 가명화** | HR·고객 데이터로 AI 학습 시 개인정보보호법 가명처리 의무 → Protegrity(정형) + Presidio(비정형 문서) 조합 |
| **데이터 플랫폼 접근 통제** | Snowflake/Databricks 기반 데이터 레이크에서 계열사·부서별 ABAC 정책 → Immuta 또는 Privacera 내장 |
| **Agent/RAG 로그 보안** | AI Agent가 사내 문서를 검색·인용할 때 PII 노출 가능 → BigID 또는 Securiti의 Agent 로그 모니터링 |
| **온프레미스 규제 데이터** | 클라우드 전송 불가 데이터는 온프레미스 Presidio 또는 Protegrity On-prem으로 처리 |
