# 구분 B: Lv.2 학습데이터 팩토리 — 개요

## 1. 구분 정의

**학습데이터 팩토리**는 AI·ML 모델 학습에 필요한 **고품질 라벨링·어노테이션 데이터**와 **합성 데이터**를 생성·관리하는 기술 영역이다.

| 하위 주제 | 코드 | 설명 |
|---|---|---|
| 라벨링·어노테이션 | B-2 | 이미지·텍스트·오디오·비디오 등 원시 데이터에 정답 레이블을 부착, AI 학습 데이터셋 구축 |
| 합성 데이터 | E-2 | 실 데이터를 통계적으로 복제하거나 AI로 생성하여 부족한 학습 데이터를 보완, 프라이버시 보호 |

두 주제는 "AI 학습 준비 데이터" 생산 관점에서 상호 보완적이다.

## 5. 선택 가이드

### 5-1. 시나리오별 추천 — 라벨링·어노테이션

| 시나리오 | 1순위 | 2순위 | 이유 |
|---|---|---|---|
| 오픈소스, 자체 호스팅 필요 | Label Studio | CVAT | Apache 2.0·MIT, 커뮤니티 강력 |
| 컴퓨터 비전, 영상 분석 특화 | Encord | CVAT | SAM3 통합, 비디오 트래킹 |
| GenAI 평가·RLHF 데이터 구축 | Label Studio | Labelbox | 대화형 RLHF, 프롬프트 평가 워크플로 |
| 로봇·자율주행, LiDAR 포인트클라우드 | Encord | CVAT | 3D 큐보이드, 센서 퓨전 |
| 의료 영상(DICOM) | Encord | Label Studio | DICOM 볼륨 MPR 지원 |
| 엔터프라이즈 멀티모달, 대규모 팀 | Labelbox | Label Studio Enterprise | 거버넌스·Alignerr 전문가 풀 |
| 비용 최소화, 소규모 프로젝트 | CVAT (OSS) | Label Studio (OSS) | 무료 오픈소스 |

### 5-2. 시나리오별 추천 — 합성 데이터

| 시나리오 | 1순위 | 2순위 | 이유 |
|---|---|---|---|
| 정형 테이블, Python 생태계 내재화 | SDV | MOSTLY AI SDK | 오픈소스, scikit-learn 스타일 |
| 텍스트+테이블 혼합 합성 | Gretel (NVIDIA) | MOSTLY AI | 텍스트 합성 지원 |
| GDPR·HIPAA 엔터프라이즈 컴플라이언스 | MOSTLY AI | Gretel | 차등 프라이버시, DHS 계약 |
| AWS/GCP 통합, 클라우드 네이티브 | Gretel (NVIDIA) | SDV Enterprise | NVIDIA 클라우드 생태계 |
| DB 익명화·합성 (SAP·Oracle 연계) | Tonic.ai | MOSTLY AI | DB 스키마 자동 탐색, 외래키 무결성 |

### 5-3. 제조업 현장 관점 코멘트

- **검사 이미지 라벨링**: 제조 공정 불량 탐지 모델 학습을 위한 결함 이미지 어노테이션에는 Encord(SAM3 AI 자동 마스킹)나 Label Studio(오픈소스, 커스텀 라벨 UI)가 적합.
- **설비 센서 데이터 합성**: 이상 탐지 모델 학습용 희귀 고장 데이터 생성에는 SDV(TimeSeriesSynthesizer) 또는 MOSTLY AI(시계열 지원)가 유효.
- **생산 DB 프라이버시 보호**: 품질 DB·MES 데이터를 개발·테스트 환경에 안전하게 공유 시 MOSTLY AI(차등 프라이버시) 또는 Tonic.ai(DB 구조 보존 합성).
- **온프레미스 필수 환경**: Label Studio OSS 또는 CVAT 자체 배포로 사내망에서 완전 운영.
- **대규모 외주 라벨링**: Labelbox의 Alignerr 전문가 풀 또는 HumanSignal의 멀티모달 데이터 서비스 활용으로 내부 인력 부담 감소.
