# Lv.2 지식그래프/온톨로지 솔루션 개요

> 작성일: 2026-06-10 | 커버 AI-Ready Data 주제: B-3 온톨로지 (제품-공정-결함-원인-조치 관계망, GraphRAG)

---

## 1. 구분 정의

Lv.2 지식그래프/온톨로지 계층은 제조·산업 데이터의 **의미론적 관계(Semantic Relationship)**를 그래프 구조로 모델링하여,
LLM/AI 시스템이 추론·질의·설명 가능한 **지식 기반(Knowledge Base)**을 구축한다.

| AI-Ready Data 주제 | 역할 |
|---|---|
| **B-3 온톨로지** | 제품-공정-결함-원인-조치 관계망 정의 및 추론 |
| **GraphRAG** | 지식그래프 + LLM 검색 결합으로 AI 정확도·설명가능성 향상 |

### 그래프 모델 유형 구분

| 모델 유형 | 특징 | 대표 솔루션 |
|---|---|---|
| **LPG (Label Property Graph)** | 노드·엣지에 속성(Property) 직접 저장, 직관적 모델링, 고성능 OLTP | Neo4j, TigerGraph, ArangoDB |
| **RDF (Resource Description Framework)** | 트리플(주어-서술어-목적어) 기반, W3C 표준, SPARQL 쿼리, 온톨로지 추론(OWL/RDFS) | GraphDB, Stardog, Amazon Neptune |
| **멀티모델** | LPG + 문서 + 키-값 + 벡터 통합 | ArangoDB, Amazon Neptune |

## 5. 선택 가이드

### 5-1. 사용 시나리오별 추천

| 시나리오 | 추천 솔루션 | 이유 |
|---|---|---|
| 개발자 생산성·빠른 배포·GraphRAG 시작 | Neo4j | 가장 넓은 생태계, 공식 GraphRAG 패키지, Cypher 직관성 |
| 표준 온톨로지(OWL/RDF) 기반 제조 지식 모델 | GraphDB | OWL 추론, SPARQL, Talk-to-Graph, Graphwise 합병으로 AI 강화 |
| AWS 생태계 + Bedrock GraphRAG 통합 | Amazon Neptune | 완전 관리형, Bedrock Knowledge Bases 직접 연동, Neptune Analytics |
| 다계열사 이종 데이터 시맨틱 통합·가상화 | Stardog | 데이터 가상화, OWL 추론, 설명가능 AI, 엔터프라이즈 거버넌스 |
| 수십억 노드 대규모 분석·딥 멀티홉 쿼리 | TigerGraph | 분산 MPP, 17.9x 성능 우위, 공급망 딥 경로 분석 |
| 멀티모델(그래프+문서+벡터) 단일 DB | ArangoDB | LPG+문서+KV+벡터 통합, Apache 2 오픈소스, AQL 단일 언어 |

### 5-2. 제조 대기업(다계열사) 관점 코멘트

**다계열사 제조 그룹** (중공업·건설기계·방산·전자 등 이종 산업) 환경에서 지식그래프 구축 시 핵심 고려사항:

1. **온톨로지 표준화 vs 빠른 시작**: 계열사 간 용어·분류체계 통일에는 RDF/OWL 기반 GraphDB·Stardog가 적합. 개별 계열사의 빠른 PoC 시작에는 Neo4j가 현실적.

2. **제품-공정-결함-원인-조치 관계망**: 제조 도메인 온톨로지(MASON, FMEA 구조 등)를 OWL로 명세하고 추론하려면 GraphDB(Graphwise)나 Stardog가 적합. LPG 기반은 온톨로지 추론에 한계.

3. **GraphRAG 구현**: 정비 문서·설비 매뉴얼·SOP를 LLM에 연결할 때 Neo4j(공식 패키지)가 가장 빠른 시작 경로. AWS 환경이라면 Neptune + Bedrock 조합이 운영 부담 최소.

4. **이종 계열사 데이터 통합**: 계열사별 ERP·MES·PLM 데이터를 물리적 이동 없이 연결하려면 Stardog(데이터 가상화)가 핵심 역량. 중장기 엔터프라이즈 지식그래프 목표에 최적.

5. **대규모 공급망 분석**: 다단계 공급망(10+ 홉) 경로 분석·이상 탐지에는 TigerGraph의 분산 MPP 아키텍처가 성능 우위(17.9x). 단, GSQL 학습 곡선 주의.

6. **운영 복잡성 최소화**: 그룹 IT 운영 조직이 없다면 Neptune(AWS 완전관리) 또는 AuraDB(Neo4j 관리형 클라우드)로 운영 부담 최소화.

7. **MCP(Model Context Protocol) 연동**: AI 에이전트 시스템과 지식그래프를 연동할 때 GraphDB 11(Graphwise)의 MCP 지원이 2025년 현재 가장 명시적.
