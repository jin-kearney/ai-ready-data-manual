# Lv.3 LLMOps·Agent Ops — AI Agent 운영 스택 개요

> 작성일: 2026-06-10 | 조사 기준: 2025~2026 최신 동향

---

## 1. 구분 정의

**LLMOps·Agent Ops**는 LLM 기반 애플리케이션과 AI 에이전트를 프로덕션 환경에서 안정적으로 운영하기 위한 관측성(Observability), 평가(Evaluation), Prompt 자산화, API/Tool 연계 표준화를 포괄하는 운영 기술 스택이다.

단순한 모델 서빙(MLOps)을 넘어, **에이전트의 추론 경로·도구 호출·멀티턴 대화를 추적**하고, **Prompt를 코드 자산처럼 버전 관리**하며, **생산 피드백을 평가 데이터로 환류**하는 전 주기를 다룬다.

## 2. AI-Ready Data 커버 주제

| 코드 | 주제 | 이 구분에서의 역할 |
|------|------|-------------------|
| **D-2** | API/Tool 연계 (MCP) | MCP 서버·레지스트리로 LLM이 호출할 수 있는 데이터·서비스 연결 표준화 |
| **D-3** | Prompt/Harness 자산화 | Prompt Registry·버전 관리·A/B 실험·템플릿 거버넌스 |
| **E-3** | AI 평가 데이터 | 평가 데이터셋 구축·LLM-as-judge·인간 어노테이션·RAG 메트릭 |
| **E-4** | Feedback Loop | 프로덕션 트레이스 → 이슈 클러스터링 → 데이터셋 보강 → 재평가 사이클 |

## 4. 솔루션 선택 가이드

### 4-1. 목적별 1차 선택

| 주 목적 | 추천 솔루션 |
|---------|------------|
| Agent 전체 관측 + Prompt 관리 통합 | LangSmith (LangChain 생태계) 또는 Langfuse (프레임워크 무관) |
| 오픈소스 완전 셀프호스팅 | Langfuse (MIT) 또는 Arize Phoenix (Apache-2.0) |
| RAG 품질 평가 전문 | Ragas (오픈소스) |
| 보안·레드팀 평가 | promptfoo (OpenAI 인수, 2026.03) |
| 멀티 LLM 비용 최적화·게이트웨이 | LiteLLM 또는 Portkey |
| MCP 기반 Tool 연계 표준화 | MCP 표준 + LiteLLM MCP Gateway |

### 4-2. 오픈소스 vs 상용

| 구분 | 오픈소스 | 상용 SaaS/Enterprise |
|------|---------|---------------------|
| 대표 솔루션 | Langfuse (MIT), Arize Phoenix (Apache-2.0), promptfoo (MIT), Ragas (MIT), LiteLLM (MIT), Portkey (오픈소스화 2026.03) | LangSmith Enterprise, Braintrust Enterprise, W&B Weave Enterprise |
| 라이선스 비용 | 무료 (인프라 비용만) | $249~$5,000+/월 (규모에 따라 협의) |
| 데이터 통제 | 완전 사내 | SaaS는 외부 전송; Enterprise VPC/온프레미스 옵션 |
| 유지보수 부담 | 내부 운영팀 필요 | 벤더 관리 |

### 4-3. 기업 환경 (보안·셀프호스팅 선호) 권고

**다계열사 공통 플랫폼 구축 관점 권고사항:**

1. **핵심 관측·평가 플랫폼**: **Langfuse** 셀프호스팅 권장  
   - MIT 라이선스, ClickHouse 기반 고성능 스토리지, OpenTelemetry 표준 준수  
   - 2026.01 ClickHouse 인수 후 인프라 안정성 강화  
   - 계열사별 프로젝트 분리·RBAC 지원

2. **LLM Gateway**: **LiteLLM** 셀프호스팅  
   - MIT 오픈소스, 100+ 모델 통합, 계열사별 API Key/Budget 격리  
   - MCP 엔드포인트 통합 지원 (2025 추가)

3. **평가 전문 도구**: **Ragas** (RAG 품질) + **promptfoo** (보안·레드팀)  
   - 둘 다 오픈소스, CI/CD 파이프라인 통합 용이

4. **MCP 표준**: 그룹 공통 Tool Registry 구성 시 MCP 서버 표준화 우선 적용  
   - 인증(OAuth 2.1), 감사 로그, 보안 등 엔터프라이즈 요건은 2026 MCP Roadmap에서 강화 중

5. **상용 고려 시점**: 에이전트 복잡도 증가, 내부 운영 인력 부족 시 LangSmith Enterprise 또는 Braintrust Enterprise 검토
