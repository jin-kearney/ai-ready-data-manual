# 기존 매뉴얼(PPTX) vs 현업판 상세목차 — 갭 분석 (A-1·A-2·B-2)

> **목적:** 기존 작성 매뉴얼(Kearney PPTX)의 실제 구성과, 내가 작성한 [현업판 상세목차 v2](05%20상세%20목차%20전체%20(20개%20주제·현업판)%20v2.md)를 항목 단위로 대조해 **현업판에 더 들어가야 할 부분**을 가려낸다.
> **판정 기준:**
> - **★ 추가(본문)** — AI-Ready에 핵심이거나 제조 현업에 직접 의미 있는 구조 요소인데 현업판에 빠졌거나 너무 얕음 → 본문 섹션/H3로 반영.
> - **○ 별첨/포인터** — 가치는 있으나 현업 눈높이엔 무거움 → 별첨 또는 한 줄 포인터로.
> - **유지** — 현업판이 의도적으로 압축/생략한 것(약어·과한 깊이) → 그대로 둠.

---

## 공통 발견 (A·B군 전반에 반복되는 3대 구조 요소)

기존 매뉴얼은 세 주제가 **같은 정본 모델**을 공유한다. 이 셋이 현업판에 약하거나 빠져 있다:

| # | 기존 매뉴얼의 공통 요소 | 현업판 반영 현황 | 판정 |
|---|---|---|---|
| 1 | **항목 5축**: Business / Technical / Compliance / Operational / **AI Data** | A-1·A-2 모두 "항목 구성"은 있으나 5축으로 명시 안 함. 특히 **AI Data 축(전처리 여부·원천 추적·AI 활용목적)**이 없음 | **★ 추가** — AI-Ready의 정체성이 AI Data 축에 있음 |
| 2 | **담당자 5역할**: 데이터 오너 / 현업 / IT / 보안 / AI 조직 (Who 독립 섹션) | 현업판은 "역할·책임"을 How 안에 한 줄로 흡수, A-1엔 사실상 없음 | **★ 추가(짧게)** — RACI는 구축의 실제 병목. 단 섹션 신설보단 각 주제 운영/구축에 1개 H3 |
| 3 | **데이터 유형별 차등 설계** (정형/시계열/문서/이미지·도면/영상·음성) | A-2·B-2에서 유형별 차이를 거의 안 다룸(제네릭) | **★ 추가** — 센서 시계열·도면 이미지는 제조 핵심. 유형별로 메타/주석 방식이 다름 |

## A-1 데이터 카탈로그

현업판 = 10섹션(개요·왜·무엇·우선순위·예시·솔루션·구축·운영·관계·지표). **커버리지 양호.** 추가할 것만:

| 기존 매뉴얼 요소 | 현업판 위치 | 판정 |
|---|---|---|
| Key Objectives 3 + 현업 인터뷰 인용(에너빌리티) | 2.1 Pain / 2.2 효과 | 유지 (인용은 계열사 예시로 활용 가능) |
| 정의·활용 3단계(전체확인·탐색·소유자) | 3·6 | 유지 |
| **항목 5축 + AI Data 축**(전처리 여부·원천 데이터 추적·AI 활용도) | 3.2~3.4 (5축·AI축 명시 안 됨) | **★ 추가** — 3장에 "AI 활용 식별 항목" H3 신설 |
| **담당자 5역할**(오너·현업·IT·보안·AI조직) | 8.4에 "역할별 기능"만 | **★ 추가** — 7(구축) 또는 8(운영)에 역할표 H3 |
| 정형 데이터 사용량 기반 Tier 선별(Backup) | 4.3 정형 중요도 선별 | 유지 (이미 반영) |
| 등록/제외 기준(정형·비정형), 취합(자동 Crawling/수동) | 4.2·4.4 | 유지 |
| 보안 검토(내부 학습 vs 외부 LLM 노출) | 4.5 | 유지 (단 "외부 LLM 노출 여부" 관점 한 줄 보강 권장) |
| 솔루션 비교(OvalEdge·Unity Catalog·AWS Glue/DataZone) | 6 솔루션 선정 | 유지 |
| 구축 프로세스(무결성 검토·To-Be·Legacy 연동) | 7 | 유지 |
| 운영: 변경관리·서비스 흐름도·Front/Back/Admin 기능 | 8 | 유지 |
| **최신화 방식: Scheduled vs Event-based**(Uber Databook) | 8.x 갱신주기만 | **○ 별첨/포인터** — "주기 갱신 vs 이벤트 갱신" 한 줄 |
| 계층 경계(A-1 / C-3 History / F-2 생애주기) | 9 | 유지 |

**A-1 결론:** 큰 누락 없음. **(1) AI Data 축 항목, (2) 담당자 역할표** 두 가지만 본문에 추가하면 충분.

## A-2 메타데이터

현업판 = 8섹션(제네릭). **가장 갭이 큼** — 기존 매뉴얼의 핵심인 "데이터 유형별 차등 설계"가 통째로 빠져 있다.

| 기존 매뉴얼 요소 | 현업판 위치 | 판정 |
|---|---|---|
| Key Objectives(오너·완결성·작성표준·도면 해설) + 인터뷰("기간 25% 소모") | 2 | 유지 |
| 정의(원천→맥락, 온도센서 예시) | 3 | 유지 |
| **메타데이터 3계층**: 테이블 / 컬럼 / 항목 상세(코드값·약어) | 3.1·3.2 (계층 명시 안 됨) | **★ 추가** — 3장에 3계층으로 재구성 |
| **항목 5축 + AI Data 축** | 3 (5축 명시 안 됨) | **★ 추가** (공통 #1) |
| **★ 데이터 유형 5종 분류**(정형 관계형 / 정형 시계열 / 비정형 문서 / 멀티미디어 / 시계열 멀티미디어) + 3대 분류요소(구조·콘텐츠·시간성) | **없음** | **★ 추가(본문 핵심)** — "데이터 유형별 메타데이터 차이" 섹션/H3 신설 |
| **유형별 특화 항목**: 시계열=Timestamp·Grain·결측구간·Sampling·Outlier / 문서=버전·언어·최신본 / 이미지·도면=해상도·구도·연관문서 / 영상=FPS·Codec·Event | **없음** | **★ 추가** — 제조(센서·도면)에 직결, 최소 핵심 항목만 |
| 담당자 5역할 | 6.3 역할만 | **★ 추가(짧게)** (공통 #2) |
| How-정의: 공통 기준 + AI 항목 기준 | 6.2 | 유지 |
| **정합성 검토 기준**(필수항목·형식·코드값 유효성·중복) + 자동 Rule 점검 | 6.2 "검증" 한 단어 | **★ 추가** — 6장에 "정합성 자동 점검" H3 |
| 정합성 보완 분담(Business·AI=현업 / Tech·Oper·Compliance=시스템 자동) | 4.2 "자동수집·사람검수" | 유지 (분담 원칙 한 줄 보강) |
| **패시브(Scheduled) vs 액티브(Event-driven) 메타데이터** 운영 | 6.3 운영(제네릭) | **○ 별첨/포인터** — "주기 갱신 vs 이벤트 자동 갱신" 한 줄 |
| 솔루션(Atlan·Unity Catalog) Backup | 6.1 도구 검토 | 유지 (A-2는 솔루션 경량 유지) |
| 계층 경계(C-3·F-2) | 7 | 유지 |
| KPI(Feedback·정비기간·완전성률) / Roadmap(변경관리·AI 자동생성) | 8 | 유지 |

**A-2 결론:** **데이터 유형별(정형/시계열/문서/이미지·도면/영상) 메타데이터 차등**이 최대 누락. 8섹션 → **유형별 설계 + 정합성 점검**을 더해 **9섹션으로 키울 것**. AI Data 축·3계층·역할도 함께.

## B-2 데이터 해설/주석

현업판 = 8섹션. 기존 매뉴얼은 **품질관리(QC) 백본**(Pilot·IAA·Gold·QA)이 두텁다 — 현업판이 "AI 1차+사람 검수"로 압축.

| 기존 매뉴얼 요소 | 현업판 위치 | 판정 |
|---|---|---|
| Key Objectives·Pain·3대효과(기계가독·훈련최적·자산화) | 2 | 유지 |
| **라벨링·주석·해설 3계층**(Macro 라벨 / Micro 주석 / Free-form 해설) | 3.1·3.2 (3계층 명시 안 됨) | **★ 추가** — 3장에 3계층 명시 |
| **Annotation 4유형**(Text/Image/Video/Audio) + 상세(NER·Bounding Box·Segmentation·Keypoint…) | **없음** | **분리 판정**: 유형별 "주석 방식이 다르다"+이미지 결함 위치(박스/영역)는 **★ 추가(얕게)** / NER·Polyline·약어는 **유지(제외)** |
| 8단계 프로세스(대상→Taxonomy→Guideline→Pilot→IAA→본라벨링→QA→버전) | 6.1·6.2·6.3에 압축 | 부분 — 아래 개별 판정 |
| Taxonomy 3원칙(상호배타·포괄·일관) + 구조(Flat/계층/Multi-label) | 6.1 taxonomy 기준 | 유지 (구조 3종은 한 줄 보강 가능) |
| **Annotation Guideline**(목적·정의·규칙·경계사례·결정규칙) + 사례집 | 3.2 주석기준·사례집 | 유지 (이미 반영) |
| **Pilot 라벨링 + IAA(작업자 간 일치도 측정)** | **없음** | **★ 추가** — "여러 사람이 같게 다는지 점검(IAA)" 핵심 QC |
| **Gold Standard Dataset(정답 기준셋)** | **없음** | **○ 별첨/포인터** — IAA 설명에 묶어 한 줄 |
| **QA·검수 + Confidence 기반 라우팅**(자동승인/검수자/전문가 SME) | 6.2 "사람 검수" | **★ 추가** — HITL 검수 단계(신뢰도별 분기) H3 |
| 버전 관리(Taxonomy·가이드·데이터셋 이력) | 3.3·6.3 | 유지 |
| 솔루션 비교(Labelbox·Scale·Label Studio·Snorkel…) | (현업판 6섹션엔 솔루션 경량) | **○ 별첨** — 도구 한 줄 |
| KPI: **IAA**·라벨오류율·처리량/비용·QA통과율·**AI Pre-label 채택률** | 8.1(정확도·합격률·생산성) | **★ 추가** — IAA(일관성)·AI 1차 라벨 채택률 |
| Roadmap 3단계(수작업70%→AI보조→자동화95%) | 8.2 고도화 | 유지 (단계 수치는 로드맵으로 구체화 가능) |

**B-2 결론:** **품질관리 절차(Pilot·IAA·QA 신뢰도 라우팅)**가 최대 누락. 이게 "AI 1차 라벨 + 사람 검수"를 실제로 신뢰 가능하게 만드는 장치다. KPI에 IAA·채택률 추가. 이미지 결함 위치 주석(박스/영역)만 얕게 보강.

## 종합 — 현업판에 반영할 우선순위 (★ 본문)

1. **[공통] AI Data 축 항목** (전처리 여부·원천 데이터 추적·AI 활용 목적) — A-1·A-2 둘 다. *AI-Ready의 정체성.*
2. **[A-2 최우선] 데이터 유형별 메타데이터 차등** (정형/시계열/문서/이미지·도면/영상) — 센서·도면 등 제조 직결. A-2를 8→9섹션.
3. **[B-2 최우선] 품질관리 절차** (Pilot·IAA 작업자 간 일치도·QA 신뢰도 라우팅) + KPI에 IAA·AI 채택률.
4. **[공통] 담당자 역할** (오너·현업·IT·보안·AI조직) — 각 주제에 짧은 H3.
5. **[A-2] 정합성 자동 점검** (필수항목·형식·코드값·중복 Rule).
6. **[A-2/B-2] 3계층 명시** — 메타데이터(테이블·컬럼·셀) / 주석(라벨·주석·해설).

## 종합 — 별첨/포인터 (○) 또는 유지

- 최신화 방식(Scheduled vs Event / 패시브 vs 액티브) — 한 줄 포인터.
- Gold Standard Dataset, 솔루션 비교 표, Annotation 세부 유형(NER·Polyline 등 약어) — 별첨 또는 제외(현업 눈높이 유지).
- 현업 인터뷰 인용은 계열사 Pain Point 예시로 재활용 가능.
