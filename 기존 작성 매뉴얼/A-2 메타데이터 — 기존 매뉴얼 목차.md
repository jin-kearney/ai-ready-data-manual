## 1. Why — 메타데이터의 필요성
1.1 Key Objectives — ① 메타데이터 오너 명확화·완결성 확보 ② 작성 표준 정의 ③ 이미지·도면 정보 해설 지원
1.2 AI Data Pain Points — 불명확한 오너십 / 작성 기준 불일치 / 도면·이미지 맥락 부재로 AI 이해 불가
   - 현업 인터뷰 인용(두산 에너빌리티·VCC) — 전체 스키마 부재, 용어·정의 모호로 프로젝트 기간 25% 소모

## 2. What — 메타데이터의 정의·구성
2.1 정의 — 원천 데이터 → 맥락 정보(메타데이터)로 활용 가능 데이터 전환
   - 5대 목적(조회·관리·AI 응용 등), 구조 유형(정형/비정형·텍스트/멀티미디어·시계열/비시계열), 온도센서 예시
2.2 메타데이터 (테이블) — 3계층: ① 테이블 메타데이터 ② 항목(Column) 메타데이터 ③ 항목 상세(코드값·약어·분류) 메타데이터
2.3 구성 기준 5축 — Business / Technical / Compliance / Operational / AI Data (축별 생성 원칙 포함)
2.4 메타데이터 유형 분류 — 3대 분류 요소(구조·콘텐츠 형태·시간성) 조합 → 5대 유형
   - ① 정형(관계형·테이블형) ② 정형(시계열) ③ 비정형(문서) ④ 비정형(멀티미디어) ⑤ 비정형(시계열 멀티미디어)
2.5 기본 구성 案 — 0. 공통 메타데이터 (Technical·Business 식별 항목)
2.6 기본 구성 案 — 0. 공통 메타데이터 (Compliance·Operational·AI 항목)
2.7 기본 구성 案 — 1. 정형 데이터(관계형·테이블형) 항목: 컬럼명·타입·Null·코드값·PK·FK·단위 등
2.8 기본 구성 案 — 2. 정형 데이터(시계열) 항목: 시계열 유형·시간 단위(Grain)·타임스탬프·결측 구간 등
2.9 기본 구성 案 — 2. 시계열 상세 Column 정보 + 3. 비정형(문서) 항목(파일형식·문서유형·언어·페이지수)
2.10 기본 구성 案 — 4. 비정형(멀티미디어) 항목 + 5. 비정형(시계열 멀티미디어) 항목(해상도·FPS·녹화유형 등)
2.11 [Backup] 데이터 Type 상세 — Enum / List / Integer / Date / String / List<Enum> 작성 기준

## 3. Who — 주요 담당자 정의
3.1 역할 5종 — 데이터 오너 / 현업 / IT / 보안 / AI 조직 (전담 부재 시 대체 지정)

## 4. How — 메타데이터 주요 업무
4.1 전체 업무 개요 — (정의) 기준 정의·유형별 특화 → (구축) 취합·현황 조사 → (운영) 변경 관리
4.2 정의 (1/2) — 메타데이터 기준 정의: 공통 기준 + AI 데이터 항목 작성 기준 (Technical·Operational 자동 수집)
4.3 정의 — 유형별 특화 ① 정형(관계형·테이블형): 대상 시스템·테이블, Key 관계(PK·FK), 컬럼별 상세 정의
4.4 정의 — 유형별 특화 ② 정형(시계열): 설비·센서 역추적, Timestamp 기준, 측정항목·단위, Sampling·Outlier 기준
4.5 정의 — 유형별 특화 ③ 비정형(문서): 문서 자산 식별·유형 분류, 생애주기·버전·최신본 확정
4.6 정의 — 유형별 특화 ④ 비정형(멀티미디어): 분류 기준, 촬영·생산 맥락, 시각·기술 품질(구도·해상도·오차)
4.7 정의 — 유형별 특화 ⑤ 비정형(시계열 멀티미디어): 영상·음성 소스 분류, 시간 구조(FPS·Codec·Timecode), 이벤트 정의
4.8 구축 — 메타데이터 취합: 크롤링/수동 수집, 구축 방향성 검토(최신화·오류·불필요 제외) 의사결정 흐름
4.9 구축 — 정합성 검토 및 구축·보완: Business·AI는 현업 보완, Technical·Operational·Compliance는 시스템 부여 / Pipeline 기획·개발
4.10 [Backup] 메타데이터 서비스 비교 — Databricks Unity Catalog / Atlan Metadata Platform
4.11 운영 — 변경 요청·검토·반영: 신규 등록(정합성 검토)·삭제·수정 기준, 타 AI 과제 영향 검토
4.12 운영 — 패시브 메타데이터(Scheduled) vs 액티브 메타데이터(Event-driven), 운영 시 전환
